# 老板/投资人版 PRD 摘要

## 1. 产品一句话
这是一个面向复杂网页流程的可观测 AI 自动化运行时，把传统脆弱的自动化脚本升级成可诊断、可优化、可批量运行、可结构化交付的执行系统。

## 2. 它解决什么问题
当前网页自动化市场长期存在三类核心问题：
1. 脚本能跑但不可维护，一改页面就整条链返工
2. 自动化是黑盒，失败后无法快速定位原因
3. AI 虽然很强，但缺少可供消费的结构化执行层，导致“AI + 自动化”大多停留在演示阶段

本产品的核心价值是把自动化从“脚本”升级为“运行时系统”，让执行、诊断、调优和交付成为一个闭环。

## 3. 产品核心能力
- 分阶段执行复杂网页流程
- 为每阶段输出统一结构化结果
- 批量运行、latest、summary、archive 归档
- 失败分类与性能瓶颈分析
- 为 AI 提供可消费的结构化 artifacts
- 支持后续接入 AI 作为诊断脑、调优脑、编排脑

## 4. 为什么现在值得做
### 市场层面
- 浏览器自动化需求一直存在，并随着业务流程在线化持续增长
- 企业和团队越来越不满足于“能跑一次”的脚本，更需要长期稳定可维护的自动化系统
- AI 大模型的普及，使“AI + 自动化”成为新需求，但目前缺少真正适合 AI 接入的执行内核

### 产品层面
当前市场主流要么是：
- 普通脚本
- 录制型 RPA
- 工作流编排
- 纯 Agent 浏览器操作

而“阶段化、可观测、可归档、可优化、适合 AI 消费”的自动化运行时并不是主流形态，存在明显产品空白。

## 5. 目标客户
### 核心客户
1. 自动化重度使用团队
- 批量网页流程执行
- 结果追踪、归档、交付需求强

2. 自动化工程团队
- 使用 Playwright/Puppeteer 等自动化框架
- 需要工程化、可维护、可诊断能力

3. AI automation builder / agent 团队
- 需要稳定的自动化执行层作为上层 AI 的专业后端

### 非核心客户
- 只想做简单录制点击的轻度用户
- 不关心长期维护的临时脚本用户

## 6. 商业价值与差异化
### 差异化
本产品不是“又一个自动化脚本工具”，而是：
- 自动化执行层
- 自动化诊断层
- AI 可消费的自动化运行时

### 相对竞争优势
- 比普通 Playwright 脚本更可观测
- 比传统 RPA 更适合复杂网页
- 比纯 Agent 操作更稳定、更可维护
- 比通用 AI 平台更懂复杂浏览器自动化运行细节

## 7. 与 OpenClaw / Claude 的关系
本产品不适合与 OpenClaw 或 Claude 在“通用 AI 平台”层面硬碰硬。
更合理的定位是：
- OpenClaw / Claude：大脑、平台、编排层
- 本产品：专业自动化执行层

这意味着产品未来有很强的集成空间，可作为 AI Agent 的浏览器自动化内核。

## 8. 商业模式方向
### 方向 A：企业/团队订阅
面向需要稳定网页自动化的团队提供：
- 运行时系统
- 批量执行
- 失败诊断
- 性能分析
- 结果交付

### 方向 B：执行引擎授权 / 私有化部署
向企业、团队、AI 平台提供自动化执行后端。

### 方向 C：AI 增值能力
高级收费模块包括：
- 自动失败诊断
- 自动调优建议
- 性能 benchmark
- 策略建议与维护辅助

### 方向 D：定制化实施服务
针对高价值复杂网页流程，提供企业定制化集成与落地。

## 9. 盈亏关键点
### 收入关键点
- 不卖“脚本”，卖“自动化基础设施”
- 不卖“会点网页”，卖“长期稳定可观测自动化”
- 不和通用 AI 平台比广度，而比自动化执行深度

### 成本关键点
- 工程化成本高，需要长期维护阶段契约和文档
- 教育市场成本高，需要 demo 让价值一眼可见
- 必须切清开源框架层与私有策略层边界

### 盈亏平衡的关键判断
只要能抓住：
- 自动化工程团队
- 重度运营自动化团队
- AI automation builder
三类用户中的任意一类，本产品都具备商业成立空间。

## 10. 当前主要风险
1. 容易被误解成“某站自动注册工具”
2. 若定位跑偏到“通用助手/通用 agent 平台”，竞争压力会明显增大
3. 若产品文档和字段契约落后于代码能力，会削弱可扩展性和可集成性

## 11. 当前最重要的执行建议
1. 继续把 staged runtime 的阶段契约和 artifact 体系收口
2. 尽快做一个一眼能看懂价值的 demo
3. 对外强调“可观测自动化运行时”，而不是“脚本能力”
4. 产品路线应优先考虑“与 AI/agent 平台集成”，而非自己重造通用平台壳子

## 12. 结论
这是一个具备明确差异化、适合往“AI-native automation runtime”方向发展的产品。

它的天花板，不在于替代单一脚本，而在于成为：
- AI Agent 的浏览器自动化执行后端
- 企业复杂网页流程的自动化基础设施
- 自动化诊断、调优、交付的一体化运行时

如果定位和 demo 做对，这个方向具备长期商业价值。
