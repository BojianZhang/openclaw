# 2026-03-21

- 记录一条 AdsPower 开发经验：`page-ops-v2.js` 导出异常时，用户看到“导出的不是相关文件”，根因可能不是文件生成错，而是后端异常时回了 HTML 错误页，前端又把它当文件下载。后续排查导出问题时，先看响应状态码、`Content-Type`、`Content-Disposition`，再看内容体是不是 HTML；对 xlsx 还要检查 zip 结构里是否有 `[Content_Types].xml`、`xl/workbook.xml`、`xl/worksheets/*.xml`。同时，导出接口异常分支要返回明确的 4xx + 文本错误，前端要阻止把错误页当文件保存。
- 再补一条：凡是用户明确要“导出为文件”的功能，默认优先做后端附件下载，不要只做前端 Blob 下载。原因是 Blob 下载在某些浏览器/宿主环境里会丢失用户可见文件名，最后下载栏显示成 UUID 或随机串，虽然内容可能是对的，但用户体验和文件识别都是错的。后续 json/txt/xlsx 导出都默认按“后端返回真实附件 + 正确文件名 + 正确 MIME”处理。
