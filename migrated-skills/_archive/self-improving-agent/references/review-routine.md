# Review Routine

## 什么时候复盘
- 开始重大任务前
- 刚完成一个复杂任务后
- 在曾经出过问题的领域继续工作时
- 周期性检查 `.learnings/`

## 复盘动作
- 看 pending 项
- 看 high priority 项
- 处理 recurring issue
- 决定是否 promote 到长期文件
- 决定是否抽成 skill

## recurring pattern
如果同类问题反复出现：
1. 搜现有 `.learnings/`
2. 建立 `See Also`
3. 提高优先级
4. 考虑写进长期规则或抽 skill

## status 取值
- `pending`
- `in_progress`
- `resolved`
- `wont_fix`
- `promoted`
- `promoted_to_skill`
