# Program-Backed Mobile Template Notes

Use this file when a template collection is not just static HTML, but a mobile site package bundled with backend code, database files, admin panels, payment hooks, user flows, or CMS-like structure.

## What makes this category different

Unlike ordinary static templates, these packages often include:
- dynamic pages
- backend admin modules
- database files
- configuration files
- payment or login integrations
- content, product, search, comment, or user-system flows

## Why this matters

These collections should not be evaluated only by homepage visuals. They should be read as:
- information architecture samples
- front-back structure references
- old integrated-site patterns
- migration or modernization case material

## What to inspect first

1. frontend entry pages
2. list/detail/search/product page types
3. admin and config structure
4. data storage and integration points
5. coupling between template and backend logic

## Practical rule

Do not mix these directly with static premium template pools at equal weight. Tag them as program-backed site cases.
