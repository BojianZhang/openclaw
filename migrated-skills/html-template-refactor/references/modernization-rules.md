# Modernization Rules

## 应保留
- 页面信息架构
- 栏目顺序
- section 节奏
- 有价值的视觉层级
- 作品/案例展示逻辑

## 应重写
- 旧 HTML 结构
- 冗余类名
- table / float 旧布局
- 旧 jQuery 交互
- 插件耦合脚本
- 不响应式的样式体系

## 应丢弃
- Flash
- 过时浏览器 hack
- 无法解释的大段内联脚本
- 只服务装饰、不服务内容的噪音动画
- 没有复用价值的旧图库/滑块实现

## 原则
- 先保留结构，再重写实现。
- 先保留模式，再丢弃旧代码。
- 不要因为“能跑”就继承脏代码。
