# MySQL 技术内幕：InnoDB 存储引擎 - 深摘要

## 这本书解决什么问题
它解决的是：**为什么 MySQL/InnoDB 在索引、事务、锁、MVCC 上会表现成现在这样。**

## 核心主线
1. 页与存储结构
2. 聚簇索引与二级索引
3. 事务与日志
4. 锁与并发控制
5. MVCC 与一致性读

## 最关键的知识骨架
### 1. 数据库性能很多时候是物理结构问题
不是 SQL 看起来对就够了，而是要理解：
- 数据怎么组织
- 索引怎么命中
- 为什么回表
- 为什么页分裂

### 2. 事务问题本质是并发控制问题
锁、MVCC、隔离级别不是概念题，而是吞吐与一致性的平衡。

### 3. Java 后端问题经常最终落到 InnoDB 行为
例如：
- 查询慢
- 死锁
- 锁等待
- 大事务拖垮系统

## 工程落地价值
### 适合回答的问题
- 聚簇索引和普通索引区别是什么？
- MVCC 为什么能减少读写冲突？
- 死锁为什么会发生？
- 为什么这个索引看起来合理但效果很差？
