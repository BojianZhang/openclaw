# MySQL 高性能 - 深摘要

## 这本书解决什么问题
它解决的是：**MySQL 为什么慢，以及如何从查询、索引、事务、架构层面把它变快。**

## 核心主线
1. 查询为什么慢
2. 索引为什么有用，以及什么时候没用
3. 事务、锁、隔离级别如何影响性能
4. 复制、分库分表、容量扩展怎么做

## 最关键的知识骨架
### 1. 先从执行计划理解 SQL
MySQL 调优优先顺序通常是：
- SQL 是否写对
- 索引是否合适
- 执行计划是否符合预期
- 是否有锁等待
- 是否有连接池/连接数问题

### 2. 索引不是越多越好
索引的代价包括：
- 写入成本
- 维护成本
- 错误索引导致优化器误判

### 3. 数据库问题经常不是“数据库单点问题”
很多时候是应用层把数据库用坏了：
- N+1 查询
- 大分页
- 事务边界过大
- 无脑 select *
- 缺少限流和缓存

## 工程落地价值
### 适合回答的问题
- 慢查询怎么查？
- 联合索引怎么设计？
- 为什么有索引还是慢？

### 后端开发的实践重点
- explain
- covering index / 回表
- 最左前缀
- 锁等待与死锁
- 批量写入与分页策略

## 常见误区
- 误以为加索引就一定快
- 误以为 ORM 生成 SQL 默认就好
- 误把数据库慢等同于 MySQL 配置有问题
