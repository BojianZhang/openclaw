# Redis 入门指南 - 深摘要

## 这本书解决什么问题
它解决的是：**Redis 能做什么、该怎么用、什么时候有收益、什么时候会引入新复杂度。**

## 核心主线
1. Redis 基本数据结构和典型用途
2. 缓存设计
3. 会话、排行榜、计数器、消息等常见场景
4. 与数据库协作时的一致性问题

## 最关键的知识骨架
### 1. Redis 首先是数据结构服务器
不是“一个更快的 MySQL”，而是：
- String
- Hash
- List
- Set
- ZSet
这些结构适合不同业务模型。

### 2. 缓存设计比“存进去”更重要
真正难的是：
- 何时失效
- 如何预热
- 如何避免穿透、击穿、雪崩
- 如何处理一致性

### 3. Redis 会把问题从“慢”换成“复杂”
它能减轻数据库压力，但会引入：
- 双写一致性
- 热点 key
- 内存淘汰
- 失效风暴

## 工程落地价值
### 适合回答的问题
- Redis 适合哪些业务场景？
- 缓存穿透/击穿/雪崩怎么办？
- Java 项目里怎么设计缓存层？
- 什么时候不该上 Redis？

### 典型实践能力
- 热点数据缓存
- 排行榜与计数器
- 限流与分布式协调的基础实现
- Cache Aside 模式

## 常见误区
- 误以为 Redis 一上性能就一定变好
- 误以为缓存一致性可以完全无成本解决
- 误以为所有数据都适合放缓存
