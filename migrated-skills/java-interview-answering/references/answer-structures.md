# Answer Structures

## 1. 概念题模板

### 短答版
- 它是什么
- 它解决什么问题
- 核心机制是什么
- 常见场景是什么

### 展开版
- 定义
- 机制
- 工程体现
- 优缺点 / 风险
- 常见误区

## 2. 对比题模板

适用于：`synchronized vs ReentrantLock`、`HashMap vs ConcurrentHashMap`、`线程 vs 线程池`、`MySQL vs Redis`。

回答结构：
1. 先给结论
2. 再分维度比较
   - 语义
   - 性能
   - 可控性
   - 适用场景
   - 风险
3. 最后说工程选择建议

## 3. 原理题模板

适用于：JMM、GC、事务、缓存一致性、HTTP keep-alive。

回答结构：
1. 先说核心目标
2. 再说关键机制
3. 再说为什么会出现问题
4. 再说在真实系统里怎么体现

## 4. 排障题模板

回答结构：
1. 现象
2. 初步怀疑点
3. 证据来源
4. 分层定位
5. 修复方案
6. 预防方案

## 5. 设计题模板

回答结构：
1. 目标
2. 约束
3. 方案
4. 权衡
5. 演进

## 6. 一分钟回答压缩法

把答案压成四句：
1. 结论
2. 原理
3. 场景
4. 风险或边界

## 7. 三分钟回答展开法

把答案拉成六段：
1. 定义
2. 背景
3. 核心机制
4. 工程场景
5. 常见坑
6. 项目例子
