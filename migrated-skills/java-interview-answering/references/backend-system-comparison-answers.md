# 后台系统样本对照答题模板

当用户在比较 CRM、进销存、云E办 这三类后台项目时，优先使用这份模板。

## 1. 进销存 vs CRM vs 云E办，最大区别是什么
- `进销存`：重点是采购、销售、库存、财务这些企业经营流程闭环
- `CRM`：重点是客户管理、营销过程、权限管理、报表统计、运营视角
- `云E办`：重点是前后端分离、权限认证、SSO、消息协同、平台工程化

## 2. 哪类项目更适合讲业务流程
- 进销存：最适合讲流程闭环
- CRM：适合讲客户生命周期和运营过程
- 云E办：业务模块可以讲，但更适合讲平台能力和协同体系

## 3. 哪类项目更适合讲工程化和协同
- 云E办最适合
- CRM / 进销存更偏传统后台业务系统表达

## 4. 哪类项目更适合讲权限体系
- 云E办：适合讲完整认证授权、安全体系、SSO
- CRM：适合讲角色权限和报表可见性
- 进销存：适合轻讲后台权限与角色管理

## 5. 默认连接句
- “这三类都属于后台系统，但问题域不一样：进销存偏经营流程，CRM偏客户运营，云E办偏前后端分离平台能力。”
- “如果问题是业务闭环，我优先讲进销存；如果问题是客户运营，我优先讲 CRM；如果问题是平台协同和安全体系，我优先讲云E办。”
