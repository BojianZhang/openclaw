# 后台系统刷题包

当用户想集中练习后台系统项目表达时，优先使用这份刷题包。覆盖三类典型样本：`进销存（JXC）`、`CRM`、`云E办（YEB）`。

## 使用方式
1. 先判断题目更偏：业务流程、客户运营、还是平台工程化
2. 选对应主样本：JXC / CRM / YEB
3. 再决定是否补权限、报表、缓存、消息或认证能力

## 第一组：共性题
1. 你做过什么后台系统项目？
2. 这类后台系统和前台电商项目有什么区别？
3. 后台系统项目最该突出的是业务流程还是技术栈？
4. 为什么后台系统项目也很值得讲？

## 第二组：进销存（JXC）专项
5. 进销存项目最核心的业务闭环是什么？
6. 采购、销售、库存、财务为什么必须连起来讲？
7. 单据流转和库存变化怎么讲才像真实业务系统？
8. 报表、权限、代码生成器为什么值得提？

## 第三组：CRM 专项
9. CRM 项目最适合突出什么价值？
10. 客户管理和营销管理怎么区分？
11. 为什么 CRM 更适合讲客户生命周期，而不是交易流程？
12. 权限和统计报表在 CRM 里为什么重要？

## 第四组：云E办（YEB）专项
13. 云E办这类项目和传统后台系统最大的区别是什么？
14. 为什么这类项目要重点讲前后端分离？
15. Spring Security / OAuth2 / JWT / Redis token 怎么区分着讲？
16. Redis 和 RabbitMQ 在办公平台里分别承接什么能力？
17. 为什么这类项目更适合讲平台工程化和协同开发？

## 第五组：对照题
18. 进销存 vs CRM vs 云E办，最大区别是什么？
19. 哪类后台项目更适合讲业务闭环？
20. 哪类后台项目更适合讲权限体系和工程化？

## 默认答案路由
- 业务闭环 -> `进销存（JXC）`
- 客户运营 / 营销 / 报表 -> `CRM`
- 前后端分离 / 权限认证 / 平台能力 -> `云E办（YEB）`
