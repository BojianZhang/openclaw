# 后台系统追问树

当用户已经能介绍后台系统项目，但还需要继续接追问时，优先使用这份追问树。

## 根问题：你做过什么后台系统项目？

### 分支 A：如果是进销存（JXC）
追问重点：
- 采购、销售、库存、财务为什么能形成闭环？
- 库存和单据流转怎么讲？
- 为什么这类项目更像企业经营流程系统？
- 报表、权限、代码生成器为什么值得提？

### 分支 B：如果是 CRM
追问重点：
- 客户管理和营销管理怎么区分？
- 客户开发 / 跟进 / 统计报表怎么讲？
- 为什么 CRM 的重点是客户生命周期而不是交易流程？
- 权限和报表为什么是核心能力？

### 分支 C：如果是云E办（YEB）
追问重点：
- 为什么它和传统后台系统不同？
- 前后端分离带来了什么价值？
- Spring Security / OAuth2 / JWT / Redis token 怎么讲？
- RabbitMQ / Redis 在平台里分别负责什么？
- 为什么这类项目更适合讲工程化、平台能力和协同开发？

## 二级判断
### 如果面试官继续问业务流程
- 优先回到进销存 / CRM

### 如果面试官继续问权限安全
- 优先回到云E办
- CRM 可补角色权限和报表可见性

### 如果面试官继续问工程化或前后端协作
- 优先回到云E办

### 如果面试官继续问后台项目的共性
统一回答：
- 都是后台系统，但问题域不同
- 进销存偏经营流程
- CRM偏客户运营
- 云E办偏前后端分离平台与安全协同
