# 基础项目 vs 复杂项目 答题模板

当用户在纠结“这个问题到底该讲基础项目还是复杂项目”时，优先使用这份模板。

## 1. 为什么有时候基础项目反而更适合回答问题

### 回答思路
因为有些问题本质上问的是基础开发能力，而不是系统规模。比如 Servlet/JSP 分层、BaseDao、Session/Cookie、前后端交互、报表统计，这些问题用基础项目反而更贴切。如果硬拿复杂项目来答，容易把问题域带偏，最后变成技术栈堆砌。

---

## 2. 为什么复杂项目不一定更有优势

### 回答思路
复杂项目的优势在于平台边界、组件协作和能力演进，但如果问题只是在问传统 Web 基础、DAO 抽象、登录态处理，那复杂项目并不会天然更有说服力。真正有说服力的是样本和问题域是否匹配。

---

## 3. 如果项目简单，怎么讲才不吃亏

### 回答思路
不要硬吹复杂度，而是把基础能力讲透。比如：为什么这样分层、BaseDao 解决了什么重复问题、Session/Cookie 怎么处理登录态、统计报表体现了什么数据组织能力。这样反而更真实，也更能体现底子。

---

## 4. 连接句
- “这类问题我更愿意用基础项目来讲，因为它更能体现底层开发能力。”
- “复杂项目更适合讲平台边界和能力演进，但这道题重点不在那儿。”
- “我会优先选和问题域最匹配的样本，而不是默认选最复杂的项目。”
