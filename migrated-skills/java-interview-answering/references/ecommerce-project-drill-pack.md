# 电商项目刷题包

当用户想集中练习电商项目表达时，优先使用这份刷题包。覆盖四类典型样本：`Ebuy`、`乐SHOP`、`易购商城`、`双11秒杀方案`。

## 使用方式
1. 先判断题目更偏：单体购物链、平台边界、传统分布式商城，还是高并发交易链
2. 选对应主样本：Ebuy / 乐SHOP / 易购商城 / 双11秒杀方案
3. 再决定是否补 Redis、MQ、搜索、Dubbo 等能力

## 第一组：共性题
1. 你做过什么电商项目？
2. 单体电商项目和商城平台项目有什么区别？
3. 电商项目里最该讲业务链，还是最该讲技术栈？
4. 为什么购物车、订单、支付值得单独讲？

## 第二组：Ebuy 专项
5. Ebuy 这类项目最适合突出什么价值？
6. 为什么它比基础项目更完整，但又不是复杂平台？
7. 购物车、订单、后台管理在单体商城里为什么值得提？

## 第三组：乐SHOP / 易购商城 专项
8. 如果让我讲一个完整商城项目，你会选哪套样本？
9. 乐SHOP 和 易购商城最大的区别是什么？
10. 乐SHOP 更适合讲哪些平台能力？
11. 易购商城为什么更适合讲 Dubbo / ZooKeeper / Solr？
12. Redis、MQ、搜索在商城平台里怎么落位？

## 第四组：秒杀专项
13. 为什么秒杀系统必须单独讲一条线？
14. Redis 和 MQ 在秒杀里分别负责什么？
15. 为什么不能拿普通电商平台硬答秒杀链？
16. 秒杀系统里限流、防刷、隐藏接口、压测该怎么讲？

## 第五组：对照题
17. 单体电商、商城平台、传统分布式商城、秒杀系统怎么区分？
18. 哪类电商样本更适合校招或初级后端？
19. 哪类电商样本更适合中级后端或电商方向岗位？
20. 哪类电商样本最适合补中间件和平台能力？

## 默认答案路由
- 单体购物链 -> `Ebuy`
- 模块化电商平台 -> `乐SHOP`
- 传统分布式商城 -> `易购商城`
- 高并发交易链 -> `双11秒杀方案`
