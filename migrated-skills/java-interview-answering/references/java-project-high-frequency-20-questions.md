# Java 项目高频问法 20 题总表

当用户想围绕不同项目样本做集中刷题、统一演练或快速选题时，优先使用这份总表。

## 使用原则
1. 先判断题目在问：基础分层、业务闭环、平台能力、微服务、传统分布式，还是高并发
2. 先选最贴的问题域样本
3. 再补该样本天然存在的中间件或平台能力

## 20 个高频项目问题

### 基础与单体项目
1. 你做过什么 Java Web 项目？
2. 这个项目的分层结构是怎样的？
3. 登录、Session、Cookie 这条线你怎么讲？
4. 为什么基础项目也值得讲？
5. 如果项目不复杂，你怎么讲出价值？

### 单体电商与后台系统
6. 购物车为什么不能只当成前端功能？
7. 订单和支付为什么值得单独讲？
8. 后台管理系统和前台购物系统有什么不同？
9. 进销存项目最核心的业务闭环是什么？
10. CRM 项目最适合突出什么价值？

### 前后端分离平台
11. 前后端分离项目和传统后台项目最大的区别是什么？
12. 权限认证体系在项目里怎么讲？
13. Spring Security / OAuth2 / JWT / Redis token 应该怎么区分着讲？
14. Redis 和 RabbitMQ 在办公平台里分别承接什么能力？

### 电商平台与传统分布式
15. 如果让我讲一个完整商城项目，你会选哪套样本？
16. 乐SHOP 和 易购商城分别适合讲什么？
17. Dubbo / ZooKeeper 在传统商城里怎么落位？
18. Redis / MQ / 搜索在电商项目里应该怎么讲？

### 微服务与高并发
19. Spring Cloud 全家桶和完整微服务项目样本有什么区别？
20. 秒杀系统为什么必须单独讲一条高并发链？

## 默认路由
- 基础单体 -> 云日记 / Ebuy
- 后台业务系统 -> 进销存 / CRM
- 前后端分离平台 -> 云E办
- 电商平台 -> 乐SHOP / 易购商城
- 微服务 -> Spring Cloud 全家桶 / 乐Z家
- 高并发 -> 双11秒杀方案
