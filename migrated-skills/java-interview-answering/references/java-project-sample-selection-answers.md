# Java 项目样本选择答题模板

当用户问“该拿哪套项目来讲这个问题”时，优先使用这份模板。

## 1. 哪套更适合讲 Spring Cloud

### 回答思路
如果重点是组件职责、调用链、网关、容错、配置中心、最小模板和阶段路线图，我会优先拿 Spring Cloud 全家桶那套资料来讲，因为它的价值更偏组件化和框架化认知，而不是完整业务平台演进。

---

## 2. 哪套更适合讲微服务项目如何逐步长出来

### 回答思路
如果重点是“一个项目怎么一版一版长服务”，我会优先拿 乐Z家，因为它更像完整微服务项目的版本演进样本，能看到推荐、详情、评论、登录、订单、消息工作流这些能力是怎么后置进入的。

---

## 3. 哪套更适合讲高并发秒杀

### 回答思路
如果重点是热点读写、Redis 快路径、MQ 削峰、反超卖、限流、验证码、隐藏接口和压测反馈，我会优先拿双11秒杀方案，因为它的问题域就是高并发交易链，而不是普通商城平台能力演进。

---

## 4. 哪套更适合讲模块化电商平台

### 回答思路
如果重点是 portal、manager、SSO、order、Redis、ES、支付、短信、云存储这些能力是怎么在一个电商平台里逐步引入的，我会优先拿 乐SHOP，因为它更像模块化平台能力演进样本。

---

## 5. 哪套更适合讲传统 Java 商城

### 回答思路
如果重点是 Dubbo + ZooKeeper、Solr 搜索、购物车、SSO、订单支付、RabbitMQ 这条传统 Java 商城路线，我会优先拿 易购商城，因为它对经典多模块商城结构和 RPC 契约层的表达更明显。

---

## 6. 如果面试官问为什么用这套样本

### 回答思路
我会说我不是随便选一个项目举例，而是按问题类型选最贴近的问题域。讲组件就用 Spring Cloud 全家桶，讲项目演进就用 乐Z家，讲高并发秒杀就用双11方案，讲模块化电商平台就用 乐SHOP，讲传统 Dubbo 商城就用 易购商城。这样能避免答非所问，也更能体现我对项目样本边界的判断。

---

## 7. 连接句
- “我会先判断问题是在问组件、项目演进、平台边界，还是高并发交易链，再决定用哪套样本来讲。”
- “不是项目越大越适合回答问题，而是样本和问题域是否匹配更重要。”
- “同样是电商项目，乐SHOP 和 易购商城 的价值点也不完全一样。”
