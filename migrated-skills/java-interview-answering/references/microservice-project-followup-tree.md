# 微服务 / Spring Cloud 追问树

当用户已经能介绍 Spring Cloud 或完整微服务项目，但还需要继续接追问时，优先使用这份追问树。

## 根问题：你做过什么微服务项目？

### 分支 A：如果是 Spring Cloud 全家桶
追问重点：
- 为什么它更适合讲组件地图，而不是完整业务项目？
- 服务发现、负载均衡、Feign、熔断、网关、配置中心怎么协同？
- Gateway、Config、Bus、Sleuth、Stream 分别负责什么？
- 为什么它适合讲治理能力和路线图？

### 分支 B：如果是 乐Z家
追问重点：
- 为什么它更适合讲完整微服务项目演进？
- 服务和模块是怎么逐步长出来的？
- 前后端协同怎么讲？
- Redis / MQ / MongoDB 等能力为什么是在后期进入？

## 二级判断
### 如果面试官继续问组件职责
- 优先回到 Spring Cloud 全家桶

### 如果面试官继续问完整项目从 0 到 1
- 优先回到 乐Z家

### 如果面试官继续问调用链协作
- 先用 Spring Cloud 解释组件关系，再用 乐Z家 补项目落地

### 如果面试官继续问微服务项目共性
统一回答：
- Spring Cloud 全家桶偏组件治理与调用链
- 乐Z家偏完整项目演进与模块增长

## 常见提醒
- 不要把 Spring Cloud 组件课讲成完整业务项目
- 不要把完整项目样本只拆成一堆组件名
