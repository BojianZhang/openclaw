# 微服务 / Spring Cloud 刷题包

当用户想集中练习 Spring Cloud、微服务项目演进、组件协作和调用链表达时，优先使用这份刷题包。覆盖两个核心样本：`Spring Cloud 全家桶`、`乐Z家`。

## 使用方式
1. 先判断题目更偏：组件地图、调用链、项目演进、还是平台能力
2. 选对应主样本：Spring Cloud 全家桶 / 乐Z家
3. 再决定是否补 Config、Gateway、注册发现、链路追踪、MQ、Redis、MongoDB 等

## 第一组：共性题
1. Spring Cloud 和完整微服务项目有什么区别？
2. 微服务项目最该讲组件，还是最该讲项目演进？
3. 为什么不能把 Spring Cloud 课程样本直接当完整项目来讲？
4. 微服务项目为什么要强调服务边界和模块增长？

## 第二组：Spring Cloud 全家桶 专项
5. Spring Cloud 全家桶最适合讲什么？
6. 服务发现、负载均衡、Feign、熔断、网关、配置中心怎么串起来？
7. Gateway、Config、Bus、Sleuth、Stream 这类组件该怎么讲职责？
8. 什么情况下应该用 Spring Cloud 样本，而不是业务项目样本？

## 第三组：乐Z家 专项
9. 乐Z家这类完整微服务项目最适合突出什么价值？
10. 为什么它比 Spring Cloud 全家桶更适合讲项目如何逐步长服务？
11. 它的前后端协同和模块演进怎么讲？
12. Redis / MQ / MongoDB 在这类项目里怎么落位？

## 第四组：对照题
13. Spring Cloud 全家桶 vs 乐Z家，最大区别是什么？
14. 哪类问题更适合用组件样本回答？
15. 哪类问题更适合用完整微服务项目样本回答？
16. 微服务项目和传统单体 / 传统分布式商城最大的区别是什么？

## 第五组：进阶题
17. 调用链里的组件如何协同工作？
18. 为什么配置中心、链路追踪和消息组件在后期才显得更重要？
19. 项目演进里，哪些能力是后期自然长出来的？
20. 如果面试官让我讲一个微服务项目，我该先讲什么、后讲什么？

## 默认答案路由
- 组件职责 / 调用链 / 路线图 -> `Spring Cloud 全家桶`
- 完整项目演进 / 模块增长 / 前后端协同 -> `乐Z家`
