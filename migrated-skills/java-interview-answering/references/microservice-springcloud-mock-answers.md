# 微服务 / Spring Cloud 模拟答法模板

当用户要围绕 Spring Cloud 或完整微服务项目做模拟面试时，优先使用这份模板。

## 1. 统一开场模板
我会把微服务方向的样本分成两类：一类是 Spring Cloud 全家桶，它更适合讲组件地图、调用链和治理能力；另一类是完整微服务项目样本，比如乐Z家，它更适合讲项目是怎么逐步长出多个服务、模块和支撑能力的。

## 2. 如果问题偏组件职责
优先切到：`Spring Cloud 全家桶`

## 3. 如果问题偏完整项目演进
优先切到：`乐Z家`

## 4. 如果问题偏调用链协作
先用 Spring Cloud 解释组件关系，再用 乐Z家 补项目落地场景。

## 5. 收束句
问组件我先讲 Spring Cloud，问项目怎么从 0 到 1 逐步长服务，我优先讲乐Z家。
