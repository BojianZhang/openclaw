# 中间件在项目样本里的落位答题模板

当用户问“Redis、RabbitMQ、Dubbo、ZooKeeper、MongoDB 这些东西在项目里到底怎么落位”时，优先使用这份模板。

## 1. Redis 在不同样本里的落位
- 秒杀：热点数据、库存快路径、限流辅助
- 乐SHOP / 易购商城：缓存商品、购物车、登录态、提升读性能
- 云E办：token、登录态、缓存热点后台数据

## 2. RabbitMQ / MQ 在不同样本里的落位
- 秒杀：削峰填谷、异步下单
- 乐SHOP：订单、通知、异步业务解耦
- 云E办：消息通知、异步任务、业务解耦

## 3. Dubbo / ZooKeeper 在不同样本里的落位
- 易购商城：传统 Java 商城的 RPC 协作、注册发现、服务治理
- 不适合拿去讲云日记、Ebuy、CRM 这类单体项目

## 4. MongoDB 在不同样本里的落位
- 乐Z家：更适合放在推荐、评论或灵活结构数据场景
- 不适合硬塞进所有项目

## 5. 回答原则
不要先背中间件名词，而要先说：这个项目有没有这个问题，这个中间件解决了什么问题，它在链路里负责哪一段。
