# 前后端分离办公平台项目追问模板

当用户已经能介绍云E办这类项目，但还需要接更深追问时，优先使用这份模板。

## 1. 为什么这类项目和传统后台管理系统不一样

### 回答思路
因为它不是单纯服务端渲染后台，而是前后端分离架构。前端负责页面路由、状态管理和交互体验，后端负责接口、认证授权、缓存、消息和业务逻辑，所以它更能体现协同开发和工程化能力。

---

## 2. 为什么安全体系值得单独讲

### 回答思路
因为办公/人事平台通常涉及登录认证、角色权限、资源访问控制、单点登录和 token 生命周期管理。Spring Security、OAuth2、JWT、Redis token 存储这些能力，不是附属功能，而是平台是否可用的关键。

---

## 3. 为什么 RabbitMQ 和 Redis 值得提

### 回答思路
Redis 通常承接缓存、登录态或热点数据，RabbitMQ 更适合承接异步通知、业务解耦和消息驱动处理。它们说明项目开始具备平台级支撑能力，而不是只停留在接口 CRUD。

---

## 4. 这类项目和 CRM / 进销存有什么区别

### 回答思路
CRM 和进销存更偏后台业务流程本身，而云E办这类项目除了业务模块，还更强调前后端分离、权限认证、消息协同和平台工程化，所以回答时要把“平台能力”讲出来，而不只是讲业务模块。

---

## 5. 连接句
- “这个项目不只是后台业务系统，更是一个前后端分离平台。”
- “我会重点讲权限认证、缓存消息和前后端协同，而不只讲页面功能。”
- “它和传统单体后台的区别，在于平台能力和工程化完整度更强。”
