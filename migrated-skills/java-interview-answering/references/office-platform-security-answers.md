# 前后端分离办公平台的权限与认证答题模板

当用户围绕云E办这类项目，被追问 Spring Security、OAuth2、JWT、Redis token、SSO 等问题时，优先使用这份模板。

## 1. 为什么权限体系值得重点讲
因为办公/人事管理平台天然涉及登录、角色、菜单、资源访问、组织边界和数据权限，这些能力决定系统是否真正可用，所以权限认证不是附属模块，而是平台核心能力。

## 2. Spring Security 在项目里的作用
它主要承担认证与授权框架能力，比如登录校验、访问控制、角色权限映射和安全拦截。项目里如果已经接入它，就不应该只说“用了个框架”，而要说它帮助把安全逻辑从业务代码里分离出来。

## 3. OAuth2 / JWT / Redis token 怎么讲
- OAuth2：更适合讲授权模型和统一认证体系
- JWT：更适合讲无状态 token 表达方式
- Redis token：更适合讲 token 状态管理、集中控制和可失效能力
回答时不要把它们混成一件事，要分清“授权协议 / token 形式 / token 存储策略”。

## 4. SSO 值得怎么讲
SSO 体现的是多个系统之间的统一登录体验和认证共享能力，说明项目已经开始从单系统登录走向平台级身份协同。

## 5. 连接句
- “这类项目里，权限认证是平台核心，不是附属功能。”
- “我会把 Security、OAuth2、JWT、Redis token 分成不同层次来讲，而不是混成一个概念。”
- “SSO 说明项目开始从单体登录能力走向平台级身份协同。”
