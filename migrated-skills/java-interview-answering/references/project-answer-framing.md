# Project Answer Framing

## 项目题万能骨架

### 1. 背景
说清业务场景、规模、角色。

### 2. 职责
说清你具体负责什么，不要把团队成果全算自己头上。

### 3. 难点
优先说真实难点：性能、并发、一致性、稳定性、治理、交付。

### 4. 决策
说清为什么这么做，而不是只说做了什么。

### 5. 结果
尽量量化：耗时下降、成功率提升、故障减少、吞吐提升。

### 6. 复盘
补一句如果重做会怎么优化。

## 项目题加分点
- 有数据
- 有取舍
- 有失败教训
- 有监控/排障/优化闭环

## 容易翻车的点
- 只讲技术名词，不讲业务目标
- 只讲结果，不讲权衡
- 把团队功劳全部说成自己做的
- 讲太满，一追问就露空
