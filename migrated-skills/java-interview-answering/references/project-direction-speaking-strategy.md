# 项目大方向总讲法策略

当用户在面试里被问“讲讲你的项目”或“不知道该先讲哪类项目”时，优先使用这份总讲法策略。

## 一句话原则
先定项目大方向，再选样本，再补中间件。顺序是：**方向 -> 样本 -> 技术点**。

## 第一步：先判断面试官在问什么
### 如果更像后台系统题
关键词：流程、权限、报表、客户、协同、后台管理
优先进入：后台系统方向

### 如果更像电商题
关键词：购物车、订单、支付、商城、搜索、秒杀、Dubbo 商城
优先进入：电商方向

### 如果更像微服务题
关键词：服务发现、网关、调用链、配置中心、Spring Cloud、服务拆分
优先进入：微服务方向

## 第二步：再在方向内部选样本
- 后台系统：进销存 / CRM / 云E办
- 电商方向：Ebuy / 乐SHOP / 易购商城 / 双11秒杀方案
- 微服务方向：Spring Cloud 全家桶 / 乐Z家

## 第三步：最后再补技术点
- 后台系统：权限、报表、代码生成器、Security、Redis、RabbitMQ
- 电商方向：Redis、MQ、Solr/ES、Dubbo、ZooKeeper、支付、短信
- 微服务方向：Gateway、Config、Feign、Sleuth、MQ、Redis、MongoDB

## 常见误区
- 一上来先背技术栈，不先定方向
- 项目方向没选清，就开始堆中间件
- 用最复杂的项目去硬答最基础的问题
- 把 Dubbo、Spring Cloud、秒杀三条线混在一套回答里

## 默认收束句
- “我会先按问题方向选项目样本，而不是默认讲最复杂的项目。”
- “如果问题偏后台系统，我讲业务闭环和权限协同；如果偏电商，我讲交易链和平台边界；如果偏微服务，我讲组件协作和项目演进。”
