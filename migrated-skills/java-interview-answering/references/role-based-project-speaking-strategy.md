# 按岗位选择项目讲法

当用户想知道不同岗位面试时应该主讲哪几套项目样本、怎么分配主次时，优先使用这份策略。

## 核心讲法
### 一句话原则
主讲最贴岗位的问题域，辅讲一个专项，保留一个基础样本兜底。

## 岗位讲法建议

### 1. 校招 / 实习
- 主讲：云日记 或 Ebuy
- 辅讲：进销存
- 讲法重点：基础能力、分层、功能闭环、业务链完整
- 不要主讲：秒杀 / 复杂微服务

### 2. 初级后端
- 主讲：Ebuy + 进销存
- 辅讲：云日记
- 讲法重点：传统 Web 基础 + 电商链 + 后台业务流

### 3. 中级后端 / 业务开发
- 主讲：乐SHOP 或 易购商城
- 辅讲：进销存 / Ebuy
- 讲法重点：模块边界、能力演进、订单支付、SSO、搜索、Redis

### 4. 微服务方向
- 主讲：Spring Cloud 全家桶 + 乐Z家
- 辅讲：乐SHOP
- 讲法重点：组件职责 + 项目演进 + 平台边界

### 5. 电商方向
- 主讲：乐SHOP + 易购商城
- 辅讲：Ebuy 或 秒杀
- 讲法重点：平台边界、购物流程、订单支付、传统路线与模块化路线

### 6. 高并发方向
- 主讲：双11秒杀方案
- 辅讲：乐SHOP
- 讲法重点：高并发链路、削峰、防刷、反超卖，不要让普通商城平台抢戏

### 7. 企业后台管理方向
- 主讲：进销存
- 辅讲：云日记
- 讲法重点：采购/销售/库存/财务/报表/系统管理

## 结构化表达建议
回答时按下面顺序：
1. 先主讲最贴岗位的一套样本
2. 面试官追问专项时，再切到辅样本
3. 如果面试官回到基础问题，就用基础样本兜底

## 常见提醒
- 不要平均介绍所有项目
- 不要把“最复杂的项目”默认当成最好的主讲项目
- 面试中能讲清 2 套项目，比说过 8 套项目更有用
