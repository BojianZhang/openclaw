# Spring Cloud 组件面试答题模板

当题目是组件单题时，优先使用这份模板。
如果题目是在问 Spring Cloud 总体是什么、第一代第二代、Alibaba、版本命名，转 `spring-cloud-foundation-answers.md`。
如果题目是在问一次请求经过哪些层、多个组件如何协作，转 `spring-cloud-call-chain-answers.md`。
如果题目需要补一句更像项目经历的表达，转 `spring-cloud-project-connection-lines.md`。

## 1. 什么是注册中心
### 短答版
注册中心可以理解成微服务里的通讯录，核心作用就是服务注册和服务发现。服务提供者启动后把自己的地址注册进去，服务消费者调用时再从注册中心拿可用实例列表，从而避免把服务地址写死。

## 2. Eureka 和 Consul 怎么比
### 短答版
Eureka 是 Netflix 体系里的经典注册中心，Spring Cloud Netflix 场景里很常见；Consul 更像一站式工具，除了注册发现，还有健康检查、KV 存储和多数据中心支持。如果团队已经希望统一做服务发现和配置管理，Consul 往往更顺手。

## 3. 什么是 Ribbon
### 短答版
Ribbon 是客户端负载均衡组件。它不是独立部署的中间件，而是运行在消费者进程里，先从注册中心拿到可用实例，再按某种策略挑一个实例发起调用。

## 4. Ribbon 和 Nginx 负载均衡区别
### 短答版
Nginx 属于服务端/集中式负载均衡，请求先到 Nginx，再由它转发；Ribbon 属于客户端负载均衡，消费者自己在本地根据服务列表选目标实例。一个偏基础设施层，一个偏微服务客户端调用层。

## 5. 什么是 Feign / OpenFeign
### 短答版
Feign 是声明式 HTTP 客户端，目的就是减少手写 RestTemplate 这类样板代码；OpenFeign 是 Spring Cloud 对 Feign 的增强版本，更自然支持 Spring MVC 注解。它提升了调用开发效率，但并不会自动解决超时、重试、幂等这些分布式问题。

## 6. 什么是 Hystrix
### 短答版
Hystrix 是 Netflix 的服务容错组件，核心目标是防止下游故障引发级联失败。它强调服务隔离、熔断、降级、请求缓存这些机制，本质上是在分布式调用里保护系统稳定性。

## 7. 什么是雪崩效应
### 短答版
雪崩效应就是一个下游服务延迟或故障后，导致上游线程大量阻塞，问题沿着依赖链不断放大，最后把整个系统拖垮。重点不只是“一个服务挂了”，而是资源耗尽和故障扩散。

## 8. Sentinel 和 Hystrix 区别
### 短答版
Hystrix 更偏远程调用隔离、熔断和 fallback；Sentinel 更偏流量治理、熔断降级、系统负载保护和控制台治理。两者都做稳定性保护，但关注重心不一样。

## 9. 为什么要用网关
### 短答版
网关是系统对外的统一入口，主要是为了隐藏内部服务结构、统一做路由和鉴权、减少客户端直接对接多个服务的复杂度。对于服务多、调用链复杂的系统，网关基本是标配。

## 10. Zuul 和 Gateway 怎么比
### 短答版
Zuul 是 Netflix 时代的经典网关方案，Gateway 是 Spring 后来主推的新一代网关方向。面试里重点别只说谁新谁旧，而是要说它们都在解决统一入口、路由、过滤、限流这些边界问题。

## 11. 什么是链路追踪 / Sleuth
### 短答版
链路追踪就是把一次请求经过的所有服务和耗时串起来。Sleuth 是 Spring Cloud 里的分布式追踪方案，常跟 Zipkin 一起用。它的价值是帮助定位调用链、耗时热点和异常路径。

## 12. Trace 和 Span 是什么
### 短答版
Trace 表示一次完整调用链，Span 表示链路中的一个调用单元。一个 Trace 里会包含多个 Span，它们通过父子关系串成一棵调用树。

## 13. 什么是 Spring Cloud Stream
### 短答版
Spring Cloud Stream 是消息驱动微服务框架，它对消息中间件做了一层抽象，帮助业务代码减少对 RabbitMQ、Kafka 这类具体中间件实现的直接耦合。

## 14. 为什么要用配置中心
### 短答版
因为微服务一多，环境配置、业务开关、外部资源地址都会变得难管理。配置中心的价值是把配置集中管理、按环境区分、支持版本控制和动态刷新，降低分布式环境下的运维成本。

## 15. Spring Cloud Config、Consul Config、Apollo 怎么看
### 短答版
Spring Cloud Config 是经典的 Spring 配置中心方案，常和 Git 结合；Consul Config 更像一站式配置与注册方案，适合已经在用 Consul 的场景；Apollo 更偏企业级配置治理平台，强调环境、集群、命名空间和实时推送，治理能力更强。
