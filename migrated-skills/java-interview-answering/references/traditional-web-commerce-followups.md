# 传统 Java Web 电商项目追问模板

当用户已经能介绍一个传统 Java Web 电商项目，但还需要接更深追问时，优先使用这份模板。

## 1. 为什么这个项目虽然是单体也值得讲

### 回答思路
因为它能体现完整业务链，而不是只做一个 CRUD 页面。商品分类、搜索、详情、最近浏览、购物车、订单、后台管理，这些功能串起来之后，已经构成了一个比较完整的电商流程样本。单体不等于没有价值，关键看业务链是否完整。

---

## 2. 为什么购物车不能只当成前端功能

### 回答思路
因为购物车本质上保存的是用户购买意图状态，它连接商品浏览和订单创建，是一个真实的后端业务状态。即使在单体项目里，也不应该只把它理解成页面按钮逻辑。

---

## 3. 为什么后台管理值得单独提

### 回答思路
因为商城不只有用户侧，后台管理代表运营侧能力。能把后台管理做出来，说明项目开始从“单纯前台展示”走向“平台具备运营维护能力”。

---

## 4. 这个项目和更复杂的商城平台有什么不同

### 回答思路
它更偏传统 Java Web 单体商城，重点在完整购物流程和前后台业务链，而不是 RPC、搜索中间件、分布式协作或复杂平台能力治理。所以我会把它讲成“业务链完整的单体商城样本”，而不是硬包装成大型分布式电商平台。

---

## 5. 连接句
- “我不会把它拔高成分布式平台，而会把它讲成传统 Java Web 电商业务链样本。”
- “这类项目最值的地方，是把购物流程和后台管理都跑通了。”
