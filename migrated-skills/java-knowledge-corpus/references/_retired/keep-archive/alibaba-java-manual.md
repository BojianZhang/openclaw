# 阿里巴巴 Java 开发手册 - 深摘要

## 这本书解决什么问题
它解决的是：**团队协作下，Java 项目如何建立统一工程规范，减少低级错误和风格混乱。**

## 核心主线
1. 命名和代码风格
2. 异常处理
3. 并发规范
4. 数据库与 SQL 规范
5. 日志与安全规范

## 最关键的知识骨架
### 1. 规范的目的不是形式主义
真正目标是：
- 降低沟通成本
- 降低低级错误率
- 提高代码审查效率
- 提高长期维护性

### 2. 规范最有价值的部分是“高频踩坑点”
比如：
- SQL 写法问题
- 日志误用
- 异常乱抛
- 并发对象误用
- 命名混乱

### 3. 团队规范需要解释“为什么”
只贴规则不解释原因，执行效果会很差。

## 工程落地价值
### 适合回答的问题
- 团队 Java 规范怎么定？
- 代码 review 有什么基本标准？
- 日志、异常、SQL、命名有哪些常见坑？

## 与其他书的连接
- 和《代码整洁之道》一起用：一个偏规范，一个偏代码风格
- 和《Effective Java》一起用：一个偏工程团队约束，一个偏 API/类设计原则
