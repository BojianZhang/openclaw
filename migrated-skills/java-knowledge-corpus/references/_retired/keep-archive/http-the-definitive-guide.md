# HTTP 权威指南 - 深摘要

## 这本书解决什么问题
它解决的是：**Web 请求到底是怎么传的，HTTP 语义是什么，代理、缓存、Cookie、连接管理为什么会影响你的 Java 后端服务。**

## 核心主线
1. HTTP 报文与语义
2. URL、方法、状态码、头部
3. 代理、缓存、Cookie、认证
4. 连接管理与性能
5. Web 故障的协议层分析

## 最关键的知识骨架
### 1. 先把 HTTP 当成协议，而不是框架接口
很多后端问题，不能只看 Spring Controller，而要回到：
- 方法语义
- 状态码语义
- Header 行为
- 缓存控制
- Keep-Alive / 连接复用

### 2. HTTP 问题经常跨层
看似“接口不稳定”，可能实际是：
- 代理问题
- 缓存问题
- Cookie / Session 问题
- 连接管理问题
- 上游/下游超时问题

### 3. 正确理解语义比背状态码更重要
比如：
- 什么时候该返回 200 / 201 / 204
- 什么时候该 400 / 401 / 403 / 404 / 409
- 幂等和安全方法是什么意思

## 工程落地价值
### 适合回答的问题
- HTTP 和 TCP 是什么关系？
- REST 接口状态码怎么设计？
- 为什么会有缓存不一致？
- 为什么接口偶发超时？
- 代理、负载均衡对请求有什么影响？

### 对 Java 后端开发的价值
- 让接口设计更规范
- 让问题排查更分层
- 能更准确地理解网关、反向代理、浏览器行为

## 常见误区
- 误把 HTTP 当成“收发 JSON 的容器”
- 误用状态码
- 误把连接问题当成业务逻辑问题

## 与其他书的连接
- 和《TCP/IP 详解》一起用：把 HTTP 问题继续下钻到传输层
- 和《MySQL 高性能》一起用：完整理解一个 Web 请求链路的瓶颈
