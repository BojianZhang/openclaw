# Java 性能优化权威指南 - 深摘要

## 这本书解决什么问题
它解决的是：**如何以工程化的方法做 Java 性能优化，而不是靠拍脑袋。**

## 核心主线
1. 明确性能目标
2. 定位真实瓶颈
3. 分层优化：JVM / 应用 / 数据库 / 网络
4. 用数据验证优化结果

## 最关键的知识骨架
### 1. 优化是一条闭环
- 先测量
- 再假设
- 再验证
- 再复盘

### 2. 系统级优化比局部微调更重要
很多时候收益最大的不是改一行 Java 代码，而是：
- SQL 优化
- 缓存设计
- 减少锁竞争
- 调整线程模型

## 工程落地价值
### 适合回答的问题
- 性能优化应该从哪下手？
- 为什么压测很好，线上却不行？
- 怎么判断优化是否真的有效？
