# Kafka / Distributed Stateful System 资料 - 深摘要

## 这组资料解决什么问题
它们解决的是：**在分布式系统里，消息流、状态管理、协调与自动化运维为什么会变复杂。**

## 核心主线
1. 消息系统与事件流
2. 状态系统的分布式管理
3. 自动化管理与系统一致性
4. 事件驱动架构与系统边界

## 最关键的知识骨架
### 1. 分布式系统的难点不只是传消息
更难的是：
- 消息顺序
- 重复消费
- 状态恢复
- 一致性与可用性权衡

### 2. 事件驱动会把耦合从同步调用转成异步状态复杂度
它能提升解耦和吞吐，但也带来更高的可观测性要求。

## 工程落地价值
### 适合回答的问题
- Kafka 为什么适合高吞吐场景？
- 事件驱动系统的状态一致性怎么理解？
- 为什么异步系统更难排障？
