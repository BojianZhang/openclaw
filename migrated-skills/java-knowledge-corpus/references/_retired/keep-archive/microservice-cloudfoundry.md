# Microservice 在 Cloud Foundry 的应用 - 深摘要

## 这份资料解决什么问题
它解决的是：**微服务不是只拆代码，还涉及部署平台、运行环境、治理方式和团队协作。**

## 核心主线
1. 微服务与平台化部署
2. 服务运行环境与工程化支撑
3. 微服务落地的组织与基础设施条件

## 最关键的知识骨架
### 1. 微服务是工程体系问题
它不只是“多几个服务”，而是：
- 配置管理
- 服务发现
- 监控
- 日志
- 部署与发布
- 环境一致性

### 2. 平台能力决定微服务落地成本
没有平台支撑，拆得越细，维护成本往往越高。

## 工程落地价值
### 适合回答的问题
- 为什么很多团队拆完微服务反而更乱？
- 微服务成功落地依赖哪些平台能力？
