# Practical API Design - 深摘要

## 这本书解决什么问题
它解决的是：**如何设计长期可演进、可扩展、对调用者友好的 API 与框架接口。**

## 核心主线
1. API 是长期契约
2. 接口设计比实现细节更重要
3. 扩展点要可控
4. 向后兼容要被当作首要约束

## 最关键的知识骨架
### 1. API 的真正用户是调用者，不是实现者自己
设计 API 时要考虑：
- 调用方式是否直观
- 是否容易误用
- 将来是否好扩展
- 升级时是否会破坏旧代码

### 2. 好 API 会主动限制错误用法
不是“什么都给”，而是“只暴露正确的能力边界”。

### 3. 框架设计的难点在于扩展性与控制力平衡
扩展点太少，用户痛苦；太多，维护者痛苦。

## 工程落地价值
### 适合回答的问题
- 公共 SDK / 工具类接口怎么设计？
- 配置 API 怎么设计更稳？
- 插件式扩展点怎么做？
- 向后兼容为什么这么重要？

## 常见误区
- 把内部实现直接暴露成外部 API
- 提前暴露太多扩展点
- 没考虑版本兼容和废弃策略
