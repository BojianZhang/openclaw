# Dubbo 的过去、现在以及未来 - 深摘要

## 这份资料解决什么问题
它解决的是：**Dubbo 在 Java 微服务 / RPC 体系中的位置，以及服务治理为什么会变得复杂。**

## 核心主线
1. RPC 的价值与代价
2. Dubbo 在服务调用中的角色
3. 服务治理、注册、路由、负载均衡
4. 微服务体系中的演进与边界

## 最关键的知识骨架
### 1. RPC 让远程调用“看起来像本地调用”，但代价不会消失
网络延迟、失败重试、序列化、版本兼容、治理成本都还在。

### 2. 服务治理是微服务复杂度的重要来源
问题不只在调用本身，还在：
- 注册发现
- 配置管理
- 流量治理
- 熔断限流
- 版本兼容
