# Java 并发编程实战 - 深摘要

## 这本书解决什么问题
它解决的是：**多线程程序为什么会错、错在哪里、如何设计得不容易错。**

## 核心主线
1. 线程安全到底是什么
2. 共享可变状态为什么危险
3. synchronized / volatile / 原子类解决的分别是什么问题
4. 如何正确使用线程池、任务、取消、中断
5. 如何在高并发系统中控制复杂度

## 最关键的知识骨架
### 1. 并发的本质不是“多开几个线程”
并发问题本质上来自：
- 共享状态
- 状态变化
- 时序不确定

### 2. 三个核心维度
- 原子性
- 可见性
- 有序性

几乎所有并发 bug 都能映射到这三个维度里理解。

### 3. 线程安全优先从“少共享”开始
比起后面疯狂加锁，更好的路线是：
- 限制共享
- 限制可变性
- 使用不可变对象
- 使用线程封闭

### 4. 锁只是手段，不是目的
锁解决的是一致性问题，但会带来：
- 性能代价
- 死锁风险
- 可扩展性问题

所以要先问：能不能不用锁，或者缩小锁粒度？

## 工程落地价值
### 适合回答的问题
- synchronized 和 volatile 区别是什么？
- 线程池怎么配置？
- 为什么会死锁？
- CompletableFuture 适合什么场景？
- 高并发系统如何减少锁竞争？

### 对后端开发最重要的章节能力
- 任务执行模型
- Executor 框架
- 取消与中断
- 并发容器
- 原子变量
- 发布与逸出

## 常见误区
- 误以为 volatile 能替代锁
- 误以为线程池越大越好
- 误以为“没有报错就线程安全”
- 误以为并发问题都是性能问题，实际上很多是正确性问题

## 排障思路
碰到并发问题时优先问：
1. 有没有共享可变状态？
2. 状态变化有没有被正确同步？
3. 是否存在发布不安全？
4. 是否有锁顺序不一致？
5. 线程池是否把系统打爆？
