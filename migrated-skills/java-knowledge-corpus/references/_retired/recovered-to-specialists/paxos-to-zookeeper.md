# 从 Paxos 到 Zookeeper - 深摘要

## 这本书解决什么问题
它解决的是：**分布式系统为什么难以保证一致性，以及 Zookeeper 这类协调系统在解决什么问题。**

## 核心主线
1. 分布式一致性问题
2. 共识与选主
3. ZooKeeper 的角色与能力
4. 注册中心、配置中心、分布式锁等场景

## 最关键的知识骨架
### 1. 分布式系统最难的不是“调用远端接口”
而是：
- 节点可能宕机
- 网络可能分区
- 消息可能延迟或乱序
- 状态可能不一致

### 2. 一致性是有成本的
强一致、可用性、分区容忍、延迟，这些不能全部白拿。

### 3. Zookeeper 不是万能工具，而是协调工具
适合：
- 命名服务
- 配置管理
- 注册发现
- 分布式锁（谨慎使用）
- 选主

## 工程落地价值
### 适合回答的问题
- 为什么微服务体系总会引入注册中心？
- 分布式锁为什么麻烦？
- 一致性问题到底难在哪？
