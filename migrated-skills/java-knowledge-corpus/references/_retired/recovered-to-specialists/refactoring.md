# 重构 - 深摘要

## 这本书解决什么问题
它解决的是：**当系统已经变乱了，如何在不推倒重来的前提下，把代码一点点改好。**

## 核心主线
1. 识别坏味道
2. 选择合适的重构手法
3. 小步修改，持续验证
4. 让代码结构更贴近真实职责

## 最关键的知识骨架
### 1. 重构不是重写
重构强调：
- 不改变外部行为
- 小步前进
- 每一步都可验证

### 2. 坏味道识别比技巧更重要
常见坏味道：
- 重复代码
- 过长函数
- 巨大类
- 条件分支泥团
- 数据泥团
- 发散式修改

### 3. 先把结构理顺，再谈优雅
重构的价值不是“变好看”，而是：
- 更容易改
- 更少副作用
- 更清晰职责边界

## 工程落地价值
### 适合回答的问题
- 这段代码怎么改更合理？
- 这个类为什么越改越难？
- 巨型 service 怎么拆？
- if/else 太多怎么收敛？

### 常用重构动作
- 提炼函数
- 搬移函数/字段
- 引入参数对象
- 分解条件表达式
- 以多态替代条件分支
- 拆分类职责

## 常见误区
- 误把大改重写叫重构
- 没测试就敢改核心逻辑
- 一次改太多，无法定位回归点

## 与其他书的连接
- 和《代码整洁之道》配合：一个讲目标风格，一个讲改造方法
- 和《Effective Java》配合：重构后的设计更稳
