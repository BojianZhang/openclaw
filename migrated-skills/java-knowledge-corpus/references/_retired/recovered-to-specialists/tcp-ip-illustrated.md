# TCP/IP 详解 - 深摘要

## 这本书解决什么问题
它解决的是：**TCP/IP 协议栈到底怎么工作，以及网络异常为什么会表现成那些奇怪现象。**

## 核心主线
1. 分层协议栈
2. IP 路由与分片
3. TCP 连接建立与关闭
4. 重传、拥塞控制、窗口机制
5. 实际报文行为分析

## 最关键的知识骨架
### 1. 网络问题必须分层看
不要把所有超时都归因到应用代码。要先分清：
- DNS
- TCP 建连
- TLS
- HTTP
- 应用逻辑

### 2. TCP 的可靠是有代价的
重传、确认、拥塞控制让它稳定，但也会带来延迟和复杂度。

### 3. 抓包是理解真实行为的关键
很多“偶发问题”其实在包级别非常明确，只是应用层日志看不出来。

## 工程落地价值
### 适合回答的问题
- 为什么会有三次握手、四次挥手？
- 为什么会重传？
- 为什么连接明明没断，应用却超时？
- 为什么服务在高延迟网络下表现很差？

## 与其他书的连接
- 和《HTTP 权威指南》一起用：把应用层和传输层打通
- 和 Wireshark 系列一起用：从理论到排障验证
