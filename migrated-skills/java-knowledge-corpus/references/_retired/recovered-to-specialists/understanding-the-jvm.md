# 深入理解 Java 虚拟机 - 深摘要

## 这本书解决什么问题
它解决的是：**Java 程序到底在 JVM 里怎么跑，以及为什么会出现 OOM、Full GC、类加载问题、性能抖动。**

## 核心主线
1. 类文件与字节码
2. 类加载机制
3. 运行时数据区
4. 对象创建与内存分配
5. 垃圾回收与内存调优
6. JVM 问题排查

## 最关键的知识骨架
### 1. 先建立“运行时地图”
要知道这些区域分别干什么：
- 堆
- 虚拟机栈
- 方法区 / 元空间
- 程序计数器
- 本地方法栈

### 2. GC 不是黑盒
GC 要从这些角度理解：
- 什么对象该活
- 什么对象可回收
- 回收时暂停多长
- 吞吐量和延迟怎么取舍

### 3. JVM 调优不是调参数竞赛
先要理解：
- 现象是什么
- 对应哪个区域出问题
- 是分配过快、回收过慢，还是对象活得太久

### 4. 类加载问题经常是“边界问题”
比如：
- 依赖冲突
- 双亲委派
- 不同 ClassLoader
- 热部署/容器环境下的类隔离

## 工程落地价值
### 适合回答的问题
- OOM 有哪几种？
- Full GC 为什么多？
- Metaspace 爆了怎么查？
- 类加载为什么会冲突？
- JVM 调优到底先看什么？

### 最重要的实战能力
- 看懂 GC 日志
- 理解堆转储和线程栈
- 区分 CPU 高、GC 频繁、锁竞争、内存泄漏这几类问题
- 建立“现象 -> 运行时区域 -> 原因”的映射

## 常见误区
- 误以为 JVM 调优就是背参数
- 误以为内存大就不会 OOM
- 误以为所有 Full GC 都该靠调大堆解决
- 误以为类加载问题只是依赖版本问题

## 排障模板
1. 先看现象：响应慢、CPU 高、内存涨、GC 频繁、线程卡住
2. 再看指标：堆、GC、线程、类加载、对象分配
3. 再确定问题层：JVM / 应用代码 / 数据库 / 网络
4. 最后才做参数和代码调整
