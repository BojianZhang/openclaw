# 章节级知识卡：Effective Java 核心章法

## 主题 1：对象创建与销毁
### 核心点
- 优先考虑静态工厂方法而不是只会用构造器
- 参数多、含义复杂时优先 Builder
- 不要滥造对象，注意复用与缓存边界
- 该释放的资源要明确释放，不要赌 GC

### 常见问题
- 构造器参数太多怎么办？
- 静态工厂比 new 好在哪？
- 对象池是不是总有价值？

## 主题 2：类与接口设计
### 核心点
- 优先组合而不是继承
- 最小暴露原则，少暴露可变状态
- 接口表达能力边界，实现隐藏在后面
- 公共 API 设计时优先考虑调用者误用成本

### 常见问题
- 什么时候该用接口？
- 继承为什么容易出问题？
- setter 为什么不该到处给？

## 主题 3：泛型 / 枚举 / 注解
### 核心点
- 泛型是类型安全工具，不是语法装饰
- 枚举适合稳定离散值与行为绑定场景
- 注解适合声明式元信息，不适合承载复杂业务逻辑

## 主题 4：异常与方法设计
### 核心点
- checked / unchecked 要基于调用者是否能恢复来判断
- 方法设计优先清晰、少歧义、可预期
- 尽早失败比延迟爆炸更好
