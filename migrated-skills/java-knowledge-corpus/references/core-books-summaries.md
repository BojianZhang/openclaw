# Core Books Summaries

> 说明：以下为基于书名、领域定位与经典知识体系整理的第一版摘要，目标是让这个 skill 具备更强的“回答问题时可调用的知识锚点”。后续可继续逐本细化。

## 1. Effective Java
**适合解决：** 如何写出更稳、更可维护、更符合最佳实践的 Java 代码。  
**核心关注：** 对象创建与销毁、泛型、枚举与注解、方法设计、异常处理、并发实践。  
**回答时可用：**
- 优先组合而不是继承
- 慎用可变对象
- 覆盖 equals/hashCode/toString 的约束
- API 设计优先清晰而不是花哨

## 2. Java 编程思想
**适合解决：** 建立完整 Java 语言心智模型。  
**核心关注：** 面向对象、集合、异常、内部类、泛型、并发、设计思想。  
**回答时可用：**
- 把语法点放回设计思想里解释
- 解释“为什么 Java 这样设计”

## 3. Java 核心技术（卷 1 / 卷 2）
**适合解决：** 系统性打基础、查 API 与语言机制。  
**核心关注：** 语言基础、标准库、GUI/高级主题、反射、网络、国际化等。  
**回答时可用：**
- 作为 Java 标准知识面补全资料
- 适合回答基础概念与 API 使用问题

## 4. Java 8 实战
**适合解决：** 理解 lambda、Stream、函数式接口、Optional、Collector。  
**核心关注：** Java 8 的编程范式变化。  
**回答时可用：**
- 解释 Stream 的惰性、链式操作、终止操作
- 区分“写起来短”和“可维护”不是一回事

## 5. Java 并发编程实战
**适合解决：** 多线程、并发安全、高并发系统设计。  
**核心关注：** happens-before、线程安全、锁、线程池、并发容器、任务取消。  
**回答时可用：**
- 先讲内存模型，再讲工具类
- 并发问题优先从共享状态和可变性分析

## 6. 深入理解 Java 虚拟机
**适合解决：** JVM 原理、类加载、内存区域、GC、调优、线上排障。  
**核心关注：** 类加载机制、字节码执行、GC 算法与收集器、JVM 调优。  
**回答时可用：**
- 解释 OOM / Full GC / 类加载异常时优先引用这套知识框架
- 不只报参数，要解释 JVM 行为

## 7. Java Performance / Java 性能优化权威指南
**适合解决：** 线上性能问题、基准测试、吞吐和延迟优化。  
**核心关注：** CPU、GC、对象分配、锁、I/O、JIT、基准测试方法。  
**回答时可用：**
- 强调“先测量再优化”
- 优化前先确认瓶颈层级：应用 / JVM / 数据库 / 网络

## 8. Head First 设计模式
**适合解决：** 设计模式入门与场景理解。  
**核心关注：** 常见 GoF 模式的意图、结构和适用场景。  
**回答时可用：**
- 讲模式时多说动机和代价，少背定义
- 区分模式是工具，不是目的

## 9. 代码整洁之道
**适合解决：** 代码风格、命名、函数设计、可读性。  
**核心关注：** 可读性、单一职责、好函数、坏味道。  
**回答时可用：**
- 对“代码怎么写更优雅”类问题给出规范化建议
- 强调一致性、可读性、边界清晰

## 10. 重构：改善既有代码设计
**适合解决：** 老代码优化、坏味道治理、结构调整。  
**核心关注：** 提炼函数、搬移职责、消除重复、简化条件逻辑。  
**回答时可用：**
- 回答“这段代码怎么改”时优先给渐进式重构路径
- 强调小步迭代，不大爆炸重写

## 11. MySQL 高性能
**适合解决：** MySQL 索引、慢查询、事务、锁、复制、容量规划。  
**核心关注：** 查询优化、索引设计、事务隔离、性能瓶颈。  
**回答时可用：**
- 先讲执行计划和索引，再讲 SQL 改法
- 后端问题优先区分是 SQL 慢、锁冲突还是连接池问题

## 12. MySQL 技术内幕：InnoDB 存储引擎
**适合解决：** InnoDB 底层原理。  
**核心关注：** B+ 树、聚簇索引、页结构、事务、MVCC、日志。  
**回答时可用：**
- 解释“为什么这个 SQL/索引会这样表现”时用底层视角分析

## 13. Redis 入门指南
**适合解决：** Redis 基础使用、缓存设计、常见数据结构。  
**核心关注：** String/List/Set/Hash/ZSet、缓存模式、发布订阅等。  
**回答时可用：**
- 回答缓存穿透/击穿/雪崩问题时提供基础策略
- 明确 Redis 适合做什么、不适合做什么

## 14. HTTP 权威指南
**适合解决：** HTTP 协议、代理、缓存、cookie、连接管理。  
**核心关注：** 请求/响应语义、头部、缓存、代理、连接复用。  
**回答时可用：**
- 解释 Web 接口问题时优先从协议语义入手
- 区分应用层问题和 TCP 层问题

## 15. UNIX 网络编程 / TCP-IP 详解 / 计算机网络
**适合解决：** TCP/UDP/Socket/连接生命周期/抓包分析。  
**核心关注：** 套接字、连接建立、传输控制、协议栈。  
**回答时可用：**
- 网络问题优先用“分层模型”分析
- Java 网络问题要映射到 TCP 行为上解释

## 16. 从 Paxos 到 Zookeeper
**适合解决：** 分布式一致性、选主、注册中心、协调系统。  
**核心关注：** 共识、一致性模型、分布式协调。  
**回答时可用：**
- 解释注册中心/配置中心/分布式锁时，优先讲一致性代价

## 17. 阿里巴巴 Java 开发手册
**适合解决：** 工程规范与团队编码约束。  
**核心关注：** 命名、异常、并发、数据库、日志、安全。  
**回答时可用：**
- 适合给“团队规范”“代码审查建议”类问题做落地规则补充
