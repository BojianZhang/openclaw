# 剑指 Offer / 程序员面试宝典 - 深摘要

## 这两类书解决什么问题
它们解决的是：**如何建立面试问题的结构化表达与常见题型框架。**

## 核心主线
1. 常见算法与数据结构题
2. Java 面试高频知识点
3. 把零散知识组织成可表达的答案

## 最关键的知识骨架
### 1. 面试不是只会做题，更重要是会表达
很多候选人不是不会，而是答得没有结构。

### 2. 面试准备要分层
- 基础语法
- 集合 / 并发 / JVM
- 数据库 / Redis / HTTP
- 项目经历与排障案例

### 3. 刷题书更适合练“输出模板”
要把知识点练成：
- 一分钟短答
- 三分钟展开
- 结合项目落地的回答

## 工程落地价值
### 适合回答的问题
- Java 面试怎么准备？
- 八股怎么答更像样？
- 算法和项目经验怎么一起准备？

## 与其他书的连接
- 它们适合做“输出训练层”，核心原理仍应来自并发、JVM、数据库、网络那些经典书
