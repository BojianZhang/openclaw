# Java 8 实战 - 深摘要

## 这本书解决什么问题
它解决的是：**Java 8 带来的函数式编程能力到底改变了什么，以及这些新写法什么时候真的有价值。**

## 核心主线
1. Lambda 表达式
2. 函数式接口
3. Stream 流水线
4. Optional
5. Collector 与数据聚合
6. 新日期时间 API

## 最关键的知识骨架
### 1. Java 8 不是单纯“语法糖升级”
它真正带来的变化是：
- 把“怎么迭代”交给框架
- 把“做什么转换/过滤/聚合”表达得更声明式
- 让集合处理方式发生变化

### 2. Stream 的价值在于表达数据处理流水线
但前提是：
- 逻辑本身适合链式表达
- 团队能读懂
- 没把简单问题写复杂

### 3. Optional 是为了显式表达“可能为空”
不是让所有 getter 都返回 Optional，也不是替代一切 null 判断。

## 工程落地价值
### 适合回答的问题
- Stream 为什么有时更优雅，有时更难读？
- map/filter/reduce 怎么理解？
- Optional 到底该怎么用？
- Java 8 对老代码重构有哪些实际价值？

## 常见误区
- 为了用 Stream 而用 Stream
- 一条链写太长，调试困难
- 把 Optional 用到实体类字段和序列化边界里

## 与其他书的连接
- 和 Effective Java 结合：让 Java 8 用法更稳
- 和重构结合：把旧式循环与条件逻辑逐步迁移为更清晰的表达
