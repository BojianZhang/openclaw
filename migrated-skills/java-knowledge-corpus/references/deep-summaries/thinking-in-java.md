# Java 编程思想 - 深摘要

## 这本书解决什么问题
它解决的是：**不仅知道 Java 怎么写，还知道 Java 为什么这样设计、对象和抽象应该怎么理解。**

## 核心主线
1. 面向对象与抽象
2. 类型系统与多态
3. 集合、异常、泛型等语言机制
4. 并发与设计思想的连接

## 最关键的知识骨架
### 1. Java 不只是语法，而是一套建模方式
写 Java 的核心不只是会写类，而是：
- 如何划分对象边界
- 如何表达职责
- 如何控制复杂度

### 2. 抽象是为了隔离复杂性
好的抽象会让调用者更轻松，差的抽象会把复杂度继续往外漏。

### 3. 语言机制背后都有设计取舍
例如：
- 泛型：类型安全与可复用
- 异常：错误边界显式化
- 多态：用统一接口表达不同实现

## 工程落地价值
### 适合回答的问题
- Java 面向对象到底该怎么理解？
- 抽象类和接口什么时候用？
- 为什么代码写着写着会越来越乱？

### 价值
它很适合把“只会写 CRUD 的 Java”提升到“更会做抽象和建模”的层次。

## 常见误区
- 误以为面向对象就是多建几个类
- 误以为抽象越多越高级
- 误以为继承天然代表复用

## 与其他书的连接
- 和《Effective Java》结合：一个给思想，一个给规则
- 和《Head First 设计模式》结合：一个讲对象协作本质，一个讲常见模式
