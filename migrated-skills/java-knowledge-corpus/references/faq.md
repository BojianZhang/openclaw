# FAQ Templates

## Q1: Java 后端应该怎么学？
回答重点：
- 先语言基础，再并发/JVM，再数据库/网络，再分布式
- 推荐路径：Java 核心技术 → Effective Java → Java 并发编程实战 → 深入理解 Java 虚拟机 → MySQL 高性能 → Redis → HTTP/TCP-IP → 重构/设计模式

## Q2: 并发问题为什么这么难排查？
回答重点：
- 难点在于共享可变状态 + 时序不确定性
- 要先看内存可见性、原子性、有序性
- 再看锁、线程池、上下文切换、阻塞点

## Q3: JVM 调优应该先看什么？
回答重点：
- 先看现象：吞吐、延迟、Full GC、OOM、CPU 飙高
- 再看证据：GC 日志、堆转储、线程栈、监控指标
- 最后再调参数，不要反过来

## Q4: MySQL 慢是因为数据库还是 Java 代码？
回答重点：
- 先分层：SQL 慢 / 索引差 / 锁等待 / 连接池耗尽 / ORM 使用不当
- 再用执行计划与链路日志定位

## Q5: Redis 一定能提升性能吗？
回答重点：
- 不是，缓存会引入一致性、失效、预热、穿透等复杂性
- 先确认热点和瓶颈，再决定是否上缓存

## Q6: 设计模式是不是越多越好？
回答重点：
- 不是，模式是对变化的响应，不是炫技工具
- 优先看问题是否真的存在：重复代码、职责混乱、扩展困难
