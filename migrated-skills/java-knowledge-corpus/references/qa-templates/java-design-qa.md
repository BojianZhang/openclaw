# 问答模板：Java 设计与代码质量

## Q: Java 代码怎么写得更规范、更可维护？
### 短答版
先抓 4 件事：职责清晰、边界明确、可变性收敛、接口稳定。Java 代码质量的核心不在语法炫技，而在抽象和长期可维护性。

### 展开版
可以从四层来看：
1. **类设计**：一个类只做一类事，避免巨大类和职责漂移。
2. **接口设计**：对外暴露稳定契约，少泄漏实现细节。
3. **可变性控制**：能不可变就尽量不可变，减少并发和状态问题。
4. **重构与规范**：通过小步重构消除重复、过长函数和坏味道。

如果要对应书本视角，主要来自《Effective Java》《代码整洁之道》《重构》《阿里巴巴 Java 开发手册》。

## Q: 什么时候该用设计模式？
### 短答版
当某个变化点已经让代码痛苦时，再用模式；不是为了显得高级而用。

### 展开版
设计模式的核心不是背名字，而是封装变化。比如：
- 规则会变：策略模式
- 创建逻辑复杂：工厂模式
- 需要叠加行为：装饰器模式
- 接口不兼容：适配器模式

先看问题是否真实存在，再决定是否引入模式。否则只会增加复杂度。
