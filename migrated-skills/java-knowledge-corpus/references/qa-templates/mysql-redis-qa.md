# 问答模板：MySQL 与 Redis

## Q: MySQL 慢查询应该怎么查？
### 短答版
先看执行计划和索引命中，再看锁、事务边界、连接池，别一上来就盲目加索引。

### 展开版
排查顺序建议：
1. 先确认真的是 SQL 层慢
2. 看 `EXPLAIN`
3. 看索引是否命中、是否回表、是否排序/分组代价大
4. 看锁等待、大事务、连接池排队
5. 再决定是改 SQL、改索引、改事务边界还是改流量策略

## Q: Redis 值不值得上？
### 短答版
有明显热点读、读多写少、允许一定不一致时值得上；否则可能只是把问题从性能转成复杂度。

### 展开版
判断 Redis 是否值得上，核心不在“能不能缓存”，而在：
- 是否真的有热点
- 一致性边界是否可接受
- 当前瓶颈是不是其实在数据库、线程池或网络

详细检查顺序优先看 `redis-checklist.md`，这里重点是回答判断逻辑。
