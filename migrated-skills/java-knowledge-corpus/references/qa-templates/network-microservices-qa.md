# 问答模板：网络与微服务

## Q: HTTP 超时应该怎么查？
### 短答版
先分层：DNS、TCP、TLS、HTTP、应用逻辑。再区分连接超时、读超时、网关超时，必要时抓包。

### 展开版
很多“HTTP 超时”其实不是一个问题，而是几类不同问题：
- 建连慢
- 下游执行慢
- 代理层超时
- 客户端主动断开
- 线程池/连接池排队

所以不能只看 controller 日志，必须看链路和层级。

## Q: 微服务到底该不该拆？
### 短答版
先问当前问题是不是架构问题；很多情况下，先把单体边界理清比直接拆微服务更划算。

### 展开版
如果问题核心已经转到服务边界、交付独立性、治理成本这些层面，再进入 `microservice-splitting-qa.md` 做更完整判断。这里保留的是“网络与微服务关联视角”，不展开微服务拆分的完整决策流程。
