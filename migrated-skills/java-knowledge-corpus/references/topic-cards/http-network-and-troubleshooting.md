# 专题卡：HTTP / 网络 / 抓包排障

## 适用问题
- HTTP 和 TCP 什么关系？
- 请求为什么超时/重传/被重置？
- 为什么服务端看着正常，客户端却报错？
- 怎么用抓包验证网络问题？

## 核心知识来源
- HTTP 权威指南
- UNIX 网络编程
- TCP/IP 详解
- 图解 TCP/IP / 计算机网络 / 网络是怎样连接的
- Wireshark 系列

## 核心结论
1. 网络问题必须分层分析：DNS / TCP / TLS / HTTP / 应用逻辑
2. HTTP 问题很多时候是连接、代理、缓存、Cookie、超时管理问题
3. socket 行为决定了很多 Java Web 框架的边界表现
4. 抓包是把“猜测”变成“证据”的关键手段

## 常见误区
- 把所有超时都归因到业务代码
- 只看应用日志不看网络层证据
- 混淆连接超时、读超时、响应超时

## 推荐回答模板
- 先判断问题在哪一层
- 再给证据抓取点
- 最后给修复建议
