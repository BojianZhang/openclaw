# 专题卡：Java 并发与多线程

## 适用问题
- synchronized / volatile / CAS 区别是什么？
- 线程池怎么配？
- 并发问题为什么难排查？
- 怎么减少锁竞争？

## 核心知识来源
- Java 并发编程实战
- 深入浅出 Java 多线程
- 深入理解 Java 虚拟机（补充内存模型背景）

## 核心结论
1. 并发问题本质来自共享可变状态和时序不确定性
2. 优先减少共享，而不是先上锁
3. volatile 解决可见性，不等于解决复合操作原子性
4. 线程池不是越大越好，要看任务类型和资源边界
5. 并发 bug 优先从原子性、可见性、有序性分析

## 常见误区
- 用 volatile 替代锁
- 线程池无限放大
- 没有报错就当线程安全
- 并发问题只盯性能，不看正确性

- 再讲工程实践和排障
