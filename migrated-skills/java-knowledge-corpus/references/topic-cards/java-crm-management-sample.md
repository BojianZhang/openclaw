# 专题卡：CRM 客户管理系统样本

## 适用问题
- 有没有适合讲客户管理/营销管理/权限管理类后台系统的样本？
- 如果项目不是商城，也不是进销存，而是 CRM 客户管理系统，该拿什么讲？
- 客户、营销、权限、报表这类后台业务系统怎么定位？

## 样本：中信 CRM
关键词：客户管理、营销管理、权限管理、客户开发、报表统计、Spring Boot、MyBatis、FreeMarker、分页

## 这套样本更适合讲什么
- CRM/客户管理系统的后台业务边界
- 客户开发、客户信息、营销流程、权限管理、报表统计等模块如何组合
- Spring Boot + MyBatis + FreeMarker 风格后台系统的典型结构
- 为什么 CRM 更适合讲“客户生命周期管理”和“运营视角”，而不是电商交易链

## 这套样本的定位
它不同于：
- 云日记：基础单体 Web 样本
- Ebuy：单体电商购物链样本
- 进销存（JXC）：采购/销售/库存/财务闭环样本

可以把它理解为：
- **客户管理系统样本**
- **营销与权限后台样本**
- **报表统计型业务系统样本**

## 更适合回答的问题
- CRM 系统一般有哪些核心模块？
- 客户管理系统怎么拆客户、营销、权限、报表？
- 为什么 CRM 项目的重点是客户生命周期和跟进过程，而不是交易流程？
- FreeMarker + Spring Boot + MyBatis 这种后台系统该怎么讲？

## 不适合讲什么
- 秒杀高并发
- 电商购物车/订单/支付链
- Spring Cloud 微服务治理
- Dubbo/RPC 分布式协作主线
