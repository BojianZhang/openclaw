# 专题卡：进销存后台管理样本

## 适用问题
- 有没有适合讲后台管理型业务系统的样本？
- 如果项目不是商城前台，而是采购、销售、库存、财务、系统管理一体的后台系统，该拿什么讲？
- 进销存项目在样本库里应该放在哪个层级？

## 样本：进销存（JXC）
关键词：系统管理、采购管理、销售管理、库存管理、财务管理、基础资料、后台管理、代码生成器、部署

## 这套样本更适合讲什么
- 后台管理型业务系统的模块边界
- 采购、销售、库存、财务这些业务链在一个管理系统里怎么组织
- Spring Boot 单体后台管理项目 + 代码生成器的项目结构
- 为什么这类项目更适合讲业务流程闭环和后台运营能力，而不是前台购物流程

## 这套样本的定位
它不同于：
- 云日记：更偏入门级单体 Web 基础项目
- Ebuy：更偏传统单体电商购物链
- 乐SHOP/易购商城：更偏商城平台/传统电商平台

可以把它理解为：
- **后台业务管理系统样本**
- **进销存流程样本**
- **单体后台运营平台样本**

## 更适合回答的问题
- 进销存系统一般有哪些核心模块？
- 后台管理系统怎么按采购/销售/库存/财务来拆？
- 为什么这类项目重点在业务流和管理能力，而不是前台体验？
- 代码生成器在管理系统里为什么常见？

## 不适合讲什么
- Spring Cloud 组件治理
- 秒杀高并发交易链
- 前台门户/购物车/搜索导向的电商体验设计
- RPC/Dubbo 为核心的分布式协作

## 常见提醒
- 不要把进销存系统讲成商城前台
- 不要忽视“采购/销售/库存/财务”这类后台业务闭环
- 它最值的地方是管理流程和业务链完整，不是中间件复杂度
