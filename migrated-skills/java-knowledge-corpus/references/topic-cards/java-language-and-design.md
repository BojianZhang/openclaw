# 专题卡：Java 语言与设计

## 适用问题
- Java 代码怎么写更规范？
- 类和接口怎么设计？
- 面向对象怎么真正落地？
- Java 8 新特性怎么合理使用？

## 核心知识来源
- Effective Java
- Java 编程思想
- Java 核心技术
- Java 8 实战
- Practical API Design
- Head First 设计模式
- 代码整洁之道
- 重构
- 阿里巴巴 Java 开发手册

## 核心结论
1. Java 开发的核心不是语法，而是抽象、边界和可维护性
2. 优先组合而不是继承
3. API 是长期契约，不要轻易暴露内部实现
4. Java 8 的 Stream / Optional 要服务于可读性，而不是炫技
5. 规范的目的不是形式主义，而是降低长期维护成本

## 常见误区
- 过度设计
- 滥用继承
- 滥用 Stream
- 接口设计不考虑向后兼容
- 重构时一次性改太多

## 推荐回答模板
- 先说设计目标
- 再说边界和约束
- 再给代码层建议
- 最后给重构/迁移路径
