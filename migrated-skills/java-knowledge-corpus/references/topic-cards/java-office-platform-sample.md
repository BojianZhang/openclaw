# 专题卡：前后端分离办公平台样本

## 适用问题
- 有没有适合讲前后端分离办公/人事/权限平台的样本？
- 如果项目同时包含 Spring Boot 后端、Vue 前端、Redis/RabbitMQ/Security/OAuth2/SSO 等配套 demo，该怎么定位？
- 云E办这类项目在样本库里属于哪一类？

## 样本：云E办（YEB）
关键词：Vue2、Element UI、Vuex、Vue Router、Axios、Spring Boot、Redis、RabbitMQ、Spring Security、OAuth2、JWT、SSO、代码生成器

## 这套样本更适合讲什么
- 前后端分离办公/人事管理平台
- Vue + Spring Boot 联动的后台管理系统
- 权限、认证、SSO、消息、缓存等配套能力如何为办公平台服务
- 为什么它比传统单体后台更强调工程化、前后端协作和安全体系

## 这套样本的定位
它不同于：
- 云日记 / Ebuy：传统 Java Web 单体
- 进销存 / CRM：后台业务系统，但前后端分离与安全体系较轻
- 乐SHOP / 易购商城：更偏电商平台

可以把它理解为：
- **前后端分离办公平台样本**
- **安全与协同能力增强的后台平台样本**
- **Vue2 + Spring Boot 管理平台样本**

## 更适合回答的问题
- 前后端分离后台项目怎么讲？
- Vue2 + Element UI + Spring Boot 的管理平台通常怎么组织？
- Spring Security / OAuth2 / JWT / Redis / RabbitMQ 在办公平台里如何落位？
- 为什么办公平台项目更适合讲权限体系、组织人事、消息通知和协同能力？

## 不适合讲什么
- 秒杀高并发主线
- Dubbo/RPC 传统分布式主线
- 纯基础 Servlet/JSP 分层入门
