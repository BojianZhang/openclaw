# 专题卡：Java 项目样本评价维度

## 适用问题
- 这些项目样本怎么用统一标准评价？
- 哪套项目更偏组件样本，哪套更偏完整项目样本？
- 哪套更适合讲传统架构，哪套更适合讲高并发？
- 样本之间怎么快速打标签？

## 统一评价维度

### 1. 问题域类型
- Spring Cloud 组件/治理
- 微服务项目演进
- 高并发交易/秒杀
- 模块化电商平台
- 传统 Java 商城

### 2. 更适合回答什么问题
看这套样本最自然地回答哪类问题，而不是覆盖面看起来有多广。

### 3. 技术时代感
- 更偏传统 Java / Dubbo / Solr / JSP-Freemarker 路线
- 更偏 Spring Boot / 模块化平台 / Redis+ES+云能力 路线
- 更偏组件教学 / 框架样本路线

### 4. 业务强度
- 偏框架/组件认知
- 偏完整业务项目
- 偏高并发交易场景

### 5. 架构复杂度
- 组件样本级
- 单项目多能力级
- 多模块平台级
- 高并发专项级

### 6. 最适合扮演的角色
- 组件样本
- 项目演进样本
- 平台边界样本
- 高并发链路样本
- 面试表达样本

## 当前样本评价

### Spring Cloud 全家桶
- 问题域：Spring Cloud 组件/治理
- 更适合：组件职责、调用链、模板、路线图
- 技术时代感：偏组件教学 / Spring Cloud 体系
- 业务强度：中，业务弱于项目样本
- 架构复杂度：组件样本级
- 角色：**组件样本 / 模板样本**

### 乐Z家
- 问题域：微服务项目演进
- 更适合：项目版本演进、模块增长、前后端协同
- 技术时代感：偏微服务项目演进样本
- 业务强度：中高
- 架构复杂度：多模块项目演进级
- 角色：**项目演进样本**

### 双11秒杀方案
- 问题域：高并发交易/秒杀
- 更适合：Redis快路径、MQ削峰、反超卖、限流、验证码、压测
- 技术时代感：偏高并发专项
- 业务强度：高，但问题域集中
- 架构复杂度：高并发专项级
- 角色：**高并发链路样本**

### 乐SHOP
- 问题域：模块化电商平台
- 更适合：portal/manager/sso/order 边界、平台能力渐进引入
- 技术时代感：偏 Spring Boot 模块化平台
- 业务强度：高
- 架构复杂度：多模块平台级
- 角色：**平台边界样本 / 平台演进样本**

### 易购商城
- 问题域：传统 Java 商城
- 更适合：Dubbo+ZooKeeper、Solr、购物车、订单支付、注册验证、RabbitMQ
- 技术时代感：偏传统 Java 商城架构
- 业务强度：高
- 架构复杂度：多模块平台级
- 角色：**传统商城样本 / RPC协作样本**

### 云日记
- 问题域：Java 入门级单体 Web 项目
- 更适合：JSP/Servlet/DAO 分层、登录态、BaseDao、基础模块拆分、统计报表
- 技术时代感：偏传统 Java Web 入门项目
- 业务强度：低到中
- 架构复杂度：单体入门项目级
- 角色：**基础能力样本 / 入门项目样本**

### Ebuy
- 问题域：传统 Java Web 电商进阶项目
- 更适合：商品分类/搜索/详情/最近浏览/购物车/订单/后台管理的完整业务链
- 技术时代感：偏传统 Java Web 单体商城
- 业务强度：中
- 架构复杂度：单体业务链项目级
- 角色：**单体电商业务链样本 / 进阶项目样本**

### 进销存（JXC）
- 问题域：后台管理型业务系统
- 更适合：采购/销售/库存/财务/系统管理这类后台流程闭环
- 技术时代感：偏 Spring Boot 单体后台管理系统
- 业务强度：中高
- 架构复杂度：单体后台业务系统级
- 角色：**后台业务闭环样本 / 管理系统样本**

### 中信 CRM
- 问题域：客户管理/营销管理后台系统
- 更适合：客户开发、营销管理、权限管理、报表统计、运营视角
- 技术时代感：偏 Spring Boot + MyBatis + FreeMarker 后台系统
- 业务强度：中高
- 架构复杂度：单体后台运营系统级
- 角色：**客户运营样本 / CRM 管理系统样本**

### 云E办（YEB）
- 问题域：前后端分离办公/人事管理平台
- 更适合：权限认证、组织人事、消息通知、缓存、前后端协作
- 技术时代感：偏 Vue2 + Element UI + Spring Boot 管理平台
- 业务强度：中高
- 架构复杂度：前后端分离后台平台级
- 角色：**办公平台样本 / 前后端协同样本**

## 快速判断规则
- 要讲组件和框架 -> 优先 Spring Cloud 全家桶
- 要讲项目怎么一版一版长 -> 优先 乐Z家
- 要讲高并发交易链 -> 优先 双11秒杀
- 要讲现代模块化电商平台边界 -> 优先 乐SHOP
- 要讲传统 Dubbo 商城链 -> 优先 易购商城

## 默认路由
- 样本总览/选样本 -> `java-project-sample-library.md`
- 需要统一评价 -> 当前专题卡
- 进入具体领域后 -> specialist / data / interview 对应层
