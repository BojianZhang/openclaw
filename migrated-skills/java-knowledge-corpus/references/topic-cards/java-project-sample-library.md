# 专题卡：Java 项目样本库

## 适用问题
- 这些 Java 项目样本分别适合讲什么？
- 哪套项目更适合讲 Spring Cloud？
- 哪套项目更适合讲传统 Dubbo 商城？
- 哪套项目更适合讲高并发秒杀？
- 面试或架构讨论时，应该拿哪套样本做例子？

## 当前样本库

### 1. 乐Z家
关键词：微服务项目演进、模块增长、前后端组合、Mongo/Redis/RabbitMQ、推荐/详情/评论/登录/订单

更适合讲：
- 微服务项目如何按版本逐步长服务
- 平台/共享/数据/业务/消息工作流模块分类
- 前后端一起长出来的项目形态
- 为什么 MQ、订单、登录这些能力是在后期进入

### 2. Spring Cloud 全家桶
关键词：Eureka、Ribbon、Feign、Hystrix、Sentinel、Zuul、Gateway、Sleuth、Stream、Config、Bus、Apollo

更适合讲：
- Spring Cloud 概念和组件地图
- 调用链、网关、容错、配置中心
- 组件职责和代际演进
- 最小模板、起步组合、阶段路线图

### 3. 双11秒杀方案
关键词：高并发、Redis、RabbitMQ、限流、验证码、隐藏接口、压测、订单链

更适合讲：
- 秒杀系统设计
- 热点读写分离
- Redis 快路径、MQ 削峰、反超卖
- 接口保护、共享 Session、压测反馈

### 4. 乐SHOP
关键词：模块化电商平台、portal/manager/sso/order/common/rpc、Redis、ES、支付、短信、云存储

更适合讲：
- 模块化电商平台能力演进
- 平台边界与支撑能力引入顺序
- Redis/ES/支付/短信/云存储在平台里的落位
- 起步组合与最小骨架

### 5. 易购商城
关键词：传统 Java 多模块商城、Dubbo、ZooKeeper、Solr、SSO、购物车、订单支付、RabbitMQ

更适合讲：
- 传统 Java 商城架构
- Dubbo + ZooKeeper + RPC 契约层
- Solr 搜索、购物车、订单支付链
- 为什么购物车是后端状态能力，为什么 RPC 要单独成层

### 6. 云日记
关键词：JSP、Servlet、MySQL、Tomcat、BaseDao、用户/类型/云记/主页/统计报表

更适合讲：
- Java 入门级单体 Web 项目
- JSP/Servlet + DAO + Service 的基础分层
- 登录态、免登录、基础功能模块拆分、统计报表
- 为什么入门项目更应该讲功能闭环和基础能力，而不是硬套复杂架构

### 7. Ebuy
关键词：JSP/Servlet、商品分类、商品搜索、商品详情、最近浏览、购物车、订单、后台管理

更适合讲：
- 传统 Java Web 电商进阶项目
- 单体商城里完整购物流程怎么逐步长出来
- 购物车、订单、后台管理在单体项目里的业务价值
- 为什么它比基础项目更完整，但又不该硬讲成复杂平台

### 8. 进销存（JXC）
关键词：系统管理、采购管理、销售管理、库存管理、财务管理、后台管理、代码生成器、部署

更适合讲：
- 后台管理型业务系统样本
- 采购/销售/库存/财务这些企业业务链如何形成闭环
- Spring Boot 单体后台管理系统的模块组织
- 为什么这类项目重点在业务流与管理能力，而不是前台购物体验

### 9. 中信 CRM
关键词：客户管理、营销管理、权限管理、客户开发、报表统计、Spring Boot、MyBatis、FreeMarker

更适合讲：
- CRM/客户管理系统样本
- 客户、营销、权限、报表这类后台模块如何组合
- 客户生命周期和运营视角下的后台系统表达
- 为什么它不该被讲成电商或进销存项目

### 10. 云E办（YEB）
关键词：Vue2、Element UI、Vuex、Vue Router、Axios、Spring Boot、Redis、RabbitMQ、Spring Security、OAuth2、JWT、SSO

更适合讲：
- 前后端分离办公/人事管理平台
- Vue + Spring Boot 管理平台协同
- 权限认证、SSO、消息、缓存等平台能力
- 为什么它比传统单体后台更强调工程化和前后端协作

## 快速选择规则

- 讲 **Spring Cloud 组件/调用链** -> 选 **Spring Cloud 全家桶**
- 讲 **微服务项目如何逐步长服务** -> 选 **乐Z家**
- 讲 **高并发秒杀/Redis+MQ/限流/防刷** -> 选 **双11秒杀方案**
- 讲 **模块化电商平台能力演进** -> 选 **乐SHOP**
- 讲 **传统 Dubbo 商城/购物车/Solr/SSO/订单支付** -> 选 **易购商城**

## 常见对照

### 乐SHOP vs 易购商城
- 乐SHOP：更适合讲模块化平台能力渐进扩展
- 易购商城：更适合讲传统 Java 商城 + Dubbo/Solr 路线

### 乐Z家 vs Spring Cloud 全家桶
- 乐Z家：更适合讲“完整项目版本演进”
- Spring Cloud 全家桶：更适合讲“组件职责、模板、路线图”

### 秒杀方案 vs 普通商城平台
- 秒杀方案：重点是高并发保护、热点读写、削峰与保护层
- 普通商城平台：重点是模块边界、业务能力、支撑能力演进

## 默认路由
- 如果问题是“该拿哪套项目当例子” -> 当前专题卡
- 如果问题进入模块边界/平台结构 -> `java-spring-microservice`
- 如果问题进入数据链、Redis/ES/MQ/订单支付 -> `java-data-backend`
- 如果问题进入面试表达 -> `java-interview-answering`

## 常见提醒
- 不要把所有项目都讲成一套套路
- 不要拿秒杀样本去硬套普通商城平台
- 不要拿组件课程样本代替完整项目演进样本
