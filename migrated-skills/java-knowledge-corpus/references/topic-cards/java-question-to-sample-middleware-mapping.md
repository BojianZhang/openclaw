# 专题卡：高频问题 -> 项目样本 -> 中间件 映射

## 适用问题
- 遇到某类面试题时，应该优先拿哪套项目样本？
- 回答这类题时，可以自然补哪些中间件？
- 某个问题域下，哪些中间件不该乱挂？

## 1. 传统 Java Web 分层 / 登录 / Session / Cookie
### 优先样本
- 云日记
- Ebuy

### 可补中间件
- 默认弱化中间件
- 如材料明确存在，可轻提缓存或搜索，但不作为主线

### 不建议乱挂
- Dubbo / Spring Cloud / 秒杀链

## 2. 单体电商购物流程 / 购物车 / 订单
### 优先样本
- Ebuy
- 易购商城
- 乐SHOP

### 可补中间件
- Redis
- MQ
- Solr/ES
- Dubbo（仅传统商城线）

### 不建议乱挂
- OAuth2 / SSO 只有材料明确存在时再展开

## 3. 后台业务闭环 / 采购销售库存财务
### 优先样本
- 进销存（JXC）

### 可补中间件
- 权限体系
- 报表统计
- 代码生成器
- 如材料明确存在，可补缓存/消息

### 不建议乱挂
- 秒杀链
- Dubbo 主线

## 4. 客户管理 / 营销管理 / 权限报表
### 优先样本
- CRM

### 可补中间件
- 权限体系
- 报表统计
- 分页
- 如材料明确存在，可补缓存

### 不建议乱挂
- 电商交易链
- Dubbo / 秒杀主线

## 5. 前后端分离后台平台 / 权限认证 / SSO
### 优先样本
- 云E办（YEB）

### 可补中间件
- Spring Security
- OAuth2
- JWT
- Redis token
- RabbitMQ
- SSO

### 不建议乱挂
- Dubbo / ZooKeeper（除非材料明确存在）
- 秒杀链 Redis+MQ 叙事

## 6. 微服务组件 / 服务发现 / 网关 / 调用链
### 优先样本
- Spring Cloud 全家桶
- 乐Z家

### 可补中间件
- Config
- Gateway
- 注册发现
- 链路追踪
- MQ
- Redis

### 不建议乱挂
- 传统单体样本
- Dubbo 线和 Spring Cloud 线混讲

## 7. 传统分布式商城 / RPC / 注册中心
### 优先样本
- 易购商城

### 可补中间件
- Dubbo
- ZooKeeper
- Solr
- Redis
- RabbitMQ

### 不建议乱挂
- Spring Cloud 组件图当主叙事

## 8. 秒杀 / 高并发 / 削峰 / 反超卖
### 优先样本
- 双11秒杀方案

### 可补中间件
- Redis
- RabbitMQ
- 限流
- 防刷
- 压测

### 不建议乱挂
- 普通后台系统样本
- 权限平台主线

## 核心原则
先判断面试问题到底在问什么，再选最贴问题域的样本；中间件只补“这个样本里天然存在的那部分”。
