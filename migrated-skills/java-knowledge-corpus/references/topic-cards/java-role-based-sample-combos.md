# 专题卡：按岗位选择项目样本组合

## 适用问题
- 不同 Java 岗位更适合主讲哪几套项目样本？
- 校招/实习/中级后端/微服务/电商/高并发方向，样本组合怎么选？
- 我不可能把所有项目都讲完，应该优先讲哪 2-3 套？

## 核心原则
岗位不同，样本组合也应该不同。
不是样本越多越好，而是**最贴岗位的问题域**最重要。

## 1. Java 后端实习 / 校招基础岗

### 推荐主样本
- 云日记
- Ebuy

### 可选辅样本
- 进销存（JXC）

### 为什么这样选
这类岗位更看重：
- 基础 Web 开发能力
- 分层思维
- 数据库和前后端交互
- 基础业务链是否完整

### 不建议主讲
- 秒杀方案
- Spring Cloud 全家桶

除非面试官明确追到高并发或微服务，否则容易显得浮。

## 2. Java 初级后端开发

### 推荐主样本
- Ebuy
- 进销存（JXC）

### 可选辅样本
- 云日记
- 乐SHOP

### 为什么这样选
这类岗位通常既看基础，也看你有没有完整业务链经验。
Ebuy 能讲购物流程，JXC 能讲后台业务闭环，组合起来很平衡。

## 3. Java 中级后端 / 业务开发岗

### 推荐主样本
- 乐SHOP
- 易购商城

### 可选辅样本
- 进销存（JXC）
- Ebuy

### 为什么这样选
这类岗位更适合讲：
- 模块边界
- 平台能力引入顺序
- 传统商城与模块化平台的实现差异
- 订单 / 支付 / SSO / 搜索 / Redis 这类支撑能力

## 4. Java 微服务 / Spring Cloud 方向

### 推荐主样本
- Spring Cloud 全家桶
- 乐Z家

### 可选辅样本
- 乐SHOP

### 为什么这样选
- Spring Cloud 全家桶 适合讲组件职责、调用链、模板、路线图
- 乐Z家 适合讲完整微服务项目怎么逐步长服务
- 乐SHOP 可补平台业务边界，不让回答只停在组件层

## 5. Java 电商业务方向

### 推荐主样本
- 乐SHOP
- 易购商城

### 可选辅样本
- Ebuy
- 双11秒杀方案

### 为什么这样选
- 乐SHOP 适合讲模块化电商平台能力演进
- 易购商城 适合讲传统 Dubbo 商城路线
- Ebuy 可补单体电商业务链
- 秒杀样本只在高并发交易链被追问时补充

## 6. Java 高并发 / 秒杀 / 性能方向

### 推荐主样本
- 双11秒杀方案

### 可选辅样本
- 乐SHOP
- Spring Cloud 全家桶

### 为什么这样选
秒杀样本是主叙事，高并发链才是核心。
乐SHOP 可补普通电商平台上下文，Spring Cloud 只能作为组件补充，不能抢主叙事。

## 7. Java 传统架构 / Dubbo 方向

### 推荐主样本
- 易购商城

### 可选辅样本
- 乐SHOP
- Ebuy

### 为什么这样选
易购商城最适合讲：
- Dubbo + ZooKeeper
- rpc 契约层
- Solr
- 购物车 / 订单 / 支付 / 注册验证

## 8. Java 企业后台管理系统方向

### 推荐主样本
- 进销存（JXC）
- 中信 CRM

### 可选辅样本
- 云E办（YEB）
- 云日记

### 为什么这样选
这类岗位更适合讲：
- 系统管理
- 采购/销售/库存/财务闭环
- 客户管理、权限、报表、运营视角
- 工程化意识与前后端分离协作（如有）

## 9. Java 前后端分离管理平台方向

### 推荐主样本
- 云E办（YEB）

### 可选辅样本
- 中信 CRM
- 进销存（JXC）

### 为什么这样选
这类岗位更适合讲：
- Vue + Spring Boot 协同
- 权限认证、SSO、JWT、Redis、RabbitMQ
- 管理平台前后端分离与工程化能力

### 默认讲法
- 主讲：云E办（强调前后端分离 + 权限安全 + 协同能力）
- 辅讲：CRM 或 进销存（补业务流程理解）
- 兜底：云日记（补基础分层能力）

## 快速选择建议

### 如果你只能重点准备 2 套样本
- 校招/基础岗：云日记 + Ebuy
- 微服务岗：Spring Cloud 全家桶 + 乐Z家
- 电商岗：乐SHOP + 易购商城
- 高并发岗：双11秒杀方案 + 乐SHOP
- 后台管理岗：进销存 + 云日记

### 如果你能准备 3 套样本
按“主样本 + 辅样本 + 兜底基础样本”组合：
- 主样本：最贴岗位方向
- 辅样本：补一个专项能力
- 兜底基础样本：云日记 或 Ebuy

## 常见提醒
- 不要为了显得厉害，校招面试硬主讲秒杀或微服务
- 不要把所有样本都平均用力
- 一般主讲 2 套，熟练掌握，效果最好
