# 专题卡：岗位 × 项目样本 × 中间件 三维选择矩阵

## 适用问题
- 某个 Java 岗位该主讲哪套项目样本？
- 讲项目时可以自然补哪些中间件？
- 哪些中间件不要为了显得高级而乱挂？

## 核心原则
先选最贴岗位的问题域样本，再补该样本天然存在的中间件能力。
顺序是：**岗位 -> 样本 -> 中间件**，不是反过来。

## 1. 校招 / 实习基础岗
### 主样本
- 云日记
- Ebuy

### 可补中间件
- 默认不强求中间件
- 如项目明确存在，可轻提 Redis / 搜索 / 简单缓存，但不能当主叙事

### 不建议乱挂
- Dubbo / ZooKeeper
- 秒杀链 Redis + MQ
- OAuth2 / SSO 平台级安全体系

### 原因
这类岗位更看基础分层、数据库、前后端交互、功能闭环，而不是中间件堆砌。

## 2. 初级后端 / 业务开发岗
### 主样本
- Ebuy
- 进销存（JXC）
- CRM

### 可补中间件
- Redis（如果项目材料里有缓存/登录态）
- 代码生成器
- 报表 / 搜索 / 权限类支撑能力

### 不建议乱挂
- Dubbo / ZooKeeper
- 秒杀高并发链

### 原因
重点是完整业务链和后台管理理解，中间件只能作为加分项，不能喧宾夺主。

## 3. 企业后台管理系统岗
### 主样本
- 进销存（JXC）
- CRM

### 可补中间件
- 权限体系
- 报表统计
- 代码生成器
- 如是前后端分离平台，可补 Redis / RabbitMQ / Security

### 不建议乱挂
- Dubbo 主线
- 秒杀链主线

### 原因
这类岗位核心是流程闭环、角色权限、运营管理和工程化，不是交易高并发。

## 4. 前后端分离管理平台岗
### 主样本
- 云E办（YEB）

### 可补中间件
- Spring Security
- OAuth2
- JWT
- Redis token
- RabbitMQ
- SSO

### 不建议乱挂
- Dubbo / ZooKeeper（除非材料明确有）
- 秒杀链 Redis+MQ 叙事

### 原因
这类岗位的重点是前后端协同、权限认证、平台支撑能力和工程化。

## 5. 电商业务岗
### 主样本
- 乐SHOP
- 易购商城
- Ebuy（作为单体补充）

### 可补中间件
- Redis
- ES / Solr
- RabbitMQ
- Dubbo / ZooKeeper（偏传统商城）
- 支付 / SSO / 短信 / 云存储 等平台能力

### 不建议乱挂
- OAuth2 / SSO 只有在项目材料明确存在时再展开
- 不要拿秒杀链代替普通商城主叙事

### 原因
这类岗位最看重交易链、平台边界和支撑能力落位。

## 6. 微服务 / Spring Cloud 岗
### 主样本
- Spring Cloud 全家桶
- 乐Z家

### 可补中间件
- 配置中心
- 网关
- 服务发现
- 链路追踪
- MQ
- Redis
- MongoDB（按样本材料）

### 不建议乱挂
- 传统单体项目样本
- Dubbo 线与 Spring Cloud 线混讲

### 原因
重点是组件职责、服务协作、项目演进和平台化能力。

## 7. 传统分布式 / Dubbo 岗
### 主样本
- 易购商城

### 可补中间件
- Dubbo
- ZooKeeper
- Solr
- Redis
- RabbitMQ

### 不建议乱挂
- Spring Cloud 组件图当主叙事

### 原因
这类岗位应该围绕传统 RPC 治理路线来讲，不要被 Spring Cloud 抢掉问题域。

## 8. 高并发 / 秒杀岗
### 主样本
- 双11秒杀方案

### 可补中间件
- Redis
- RabbitMQ
- 限流 / 防刷 / 隐藏接口 / 压测

### 不建议乱挂
- 用普通后台系统样本硬答高并发交易链
- 用 OAuth2 / SSO 这类平台安全能力代替秒杀主线

### 原因
重点是高并发链路、保护策略和异步削峰。

## 快速口诀
- 基础岗：先项目闭环，少讲中间件
- 后台岗：先业务流程，再讲权限/报表/工程化
- 前后端分离岗：先平台协同，再讲安全/缓存/消息
- 电商岗：先交易链和平台边界，再讲 Redis/MQ/搜索
- 微服务岗：先组件协作和演进，再讲中间件落位
- 秒杀岗：先高并发链路，再讲 Redis+MQ

## 最终提醒
中间件不是“高级感装饰品”。
只有当项目样本天然需要它、岗位问题域也需要它时，才应该展开去讲。
