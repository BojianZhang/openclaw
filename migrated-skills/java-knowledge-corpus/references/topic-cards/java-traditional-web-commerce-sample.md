# 专题卡：传统 Java Web 电商进阶样本

## 适用问题
- 有没有介于“入门级单体项目”和“复杂商城平台”之间的样本？
- 如果我的项目是传统 Java Web 电商，不是微服务，也不是纯基础练手项目，该拿什么讲？
- 购物车、订单、后台管理这些功能在传统单体商城里怎么逐步长出来？

## 样本：Ebuy
关键词：JSP/Servlet、商品分类、商品搜索、商品详情、最近浏览、购物车、订单管理、后台管理

## 这套样本更适合讲什么
- 传统 Java Web 电商项目的逐步功能扩展
- 从展示导航栏/商品分类，到注册登录、搜索、详情、最近浏览、购物车、订单、后台管理的完整链路
- 单体商城里前台购物流程与后台管理流程的共存
- 为什么购物车、订单、后台管理在传统 Web 电商项目里值得单独讲

## 这套样本的定位
它比“云日记”更偏业务平台，适合讲完整购物流程；
它又比“乐SHOP/易购商城”更轻，不强调 RPC、Dubbo、搜索中间件、支付生态等复杂平台能力。

可以把它理解为：
- **入门项目之上** 的传统 Java Web 电商样本
- **复杂商城平台之下** 的业务链样本

## 不适合讲什么
- Spring Cloud 组件治理
- Dubbo + ZooKeeper 服务协作
- Redis/MQ/秒杀高并发链路
- 平台级多模块能力治理

## 快速定位
如果问题是：
- “传统 Java Web 商城项目怎么讲？”
- “购物车、订单、后台管理这类功能在单体项目里怎么组织？”
- “有没有比云日记更完整、但又不至于上升到复杂平台的样本？”

优先用这套样本。

## 常见提醒
- 不要把它硬讲成分布式商城
- 不要因为它是单体，就忽视购物车/订单/后台管理这些真实业务链价值
- 它最适合证明：传统 Java Web 也可以跑通完整购物流程
