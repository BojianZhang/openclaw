# 专题卡：JVM 与性能优化

## 适用问题
- OOM / Full GC 怎么查？
- JVM 调优从哪开始？
- Java 服务慢怎么排？
- 为什么压测和线上表现不一致？

## 核心知识来源
- 深入理解 Java 虚拟机
- 实战 Java 虚拟机
- Java Performance
- Java 性能优化权威指南

## 核心结论
1. JVM 与性能问题必须分层看，不能只盯参数。
2. 性能问题要区分 CPU、内存、GC、锁、I/O、数据库、网络。
3. 吞吐量和延迟常常需要权衡。
4. 先测量，再优化；先定位层级，再做动作。
5. Full GC、抖动、OOM 往往只是表象，要回到对象分配、生命周期和运行时区域理解。

## 常见误区
- 一上来改 JVM 参数
- CPU 不高就觉得没性能问题
- 只盯 Java 代码，不看数据库/网络
- 优化没有基线和验证
