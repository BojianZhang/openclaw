# 专题卡：微服务基础认知

如果问题是在问“什么是微服务、为什么会出现、和 SOA/单体/分布式怎么区分”，优先用这张卡。
如果问题已经进入“该不该拆、边界怎么定、分布式复杂度怎么判断”，转 `microservices-and-distributed-systems.md`。
如果问题已经进入 Spring Cloud 平台总览，转 `spring-cloud-foundations.md`。

## 适用问题
- 什么是微服务？
- 单体、分布式、SOA、微服务之间怎么串起来理解？
- Dubbo 和 Spring Cloud 怎么比较？
- 为什么微服务强调无状态、RESTful、前后端分离？
- AKF 拆分原则怎么解释？

## 核心知识点
1. 架构演进不是为了追时髦，而是为了解决规模、协作、交付与治理问题
2. 微服务本质是围绕业务能力拆分、独立部署、轻量通信、边界自治
3. SOA 与微服务的差别主要在粒度、边界、数据归属、治理方式
4. Dubbo vs Spring Cloud 不只是 RPC vs HTTP，而是服务调用风格与生态重心不同
5. 无状态、RESTful、前后端分离都属于“易扩展、易协作、易治理”的工程原则，不是微服务定义本身
6. AKF 的 X/Y/Z 轴说明：服务拆分只是扩展手段之一，不要把 Y 轴当成唯一解

## 常见提醒
- 不要把“微服务”回答成纯技术名词堆砌
- 不要只讲优点，不讲分布式复杂度
- 不要把前后端分离当成微服务的定义
- 不要把 RPC 当成本地调用的无损替代
