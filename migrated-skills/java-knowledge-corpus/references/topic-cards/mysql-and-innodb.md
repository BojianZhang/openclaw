# 专题卡：MySQL 与 InnoDB

## 适用问题
- 慢查询怎么查？
- 联合索引怎么设计？
- MVCC 怎么理解？
- 为什么会锁等待/死锁？

## 核心知识来源
- MySQL 高性能
- MySQL 技术内幕：InnoDB 存储引擎

## 核心结论
1. MySQL 问题优先看执行计划、索引、锁、事务边界
2. 索引不是越多越好，关键是是否匹配访问模式
3. InnoDB 行为要从页结构、聚簇索引、MVCC、日志理解
4. 很多数据库慢其实是应用把数据库用坏了

## 常见误区
- 觉得有索引就一定快
- 把 ORM 生成 SQL 当黑盒
- 大事务、N+1、无脑 select *
- 忽视连接池和锁等待
