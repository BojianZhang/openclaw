# 专题卡：项目大方向总对照

## 适用问题
- 后台系统、电商项目、微服务项目三大方向怎么区分？
- 面试官让我讲项目时，我应该先往哪一类项目上靠？
- 不同项目大方向各自最适合讲什么？

## 1. 后台系统方向
### 代表样本
- 进销存（JXC）
- CRM
- 云E办（YEB）

### 更适合讲什么
- 企业经营流程
- 客户运营与权限报表
- 前后端分离平台能力
- 后台管理、组织协同、工程化

### 不该主讲什么
- 秒杀高并发链
- 传统 Dubbo 商城主线
- Spring Cloud 组件治理主线

## 2. 电商项目方向
### 代表样本
- Ebuy
- 乐SHOP
- 易购商城
- 双11秒杀方案

### 更适合讲什么
- 购物链路
- 平台边界
- 订单/支付/购物车/搜索
- 传统分布式商城
- 高并发交易链

### 不该主讲什么
- 把普通电商项目讲成后台管理系统
- 把秒杀链当成所有电商项目的默认叙事

## 3. 微服务项目方向
### 代表样本
- Spring Cloud 全家桶
- 乐Z家

### 更适合讲什么
- 组件地图
- 调用链协作
- 服务发现/网关/配置中心/链路追踪
- 完整微服务项目演进
- 模块增长与支撑能力后期引入

### 不该主讲什么
- 把 Spring Cloud 课程样本直接当完整业务项目
- 把完整项目只讲成组件名清单

## 快速判断
- 问业务闭环、后台流程、客户运营、权限协同 -> 后台系统方向
- 问购物流程、商城平台、订单支付、Dubbo 商城、秒杀 -> 电商方向
- 问组件治理、调用链、服务拆分、微服务演进 -> 微服务方向

## 总原则
先判断面试问题的业务域和架构域，再决定进入哪条项目主线；不要先想“我最熟哪个项目”，而要先想“这个问题最像哪一类项目”。
