# 专题卡：Redis 与缓存设计

## 适用问题
- Redis 适合做什么？
- 缓存穿透/击穿/雪崩怎么处理？
- Java 项目怎么设计缓存层？
- 什么时候不该用 Redis？

## 核心知识来源
- Redis 入门指南
- MySQL 高性能（数据库与缓存协同视角）

## 核心结论
1. Redis 首先是数据结构服务器，不是“更快的数据库替代品”
2. 缓存设计比“把数据塞进去”更重要
3. Redis 会把部分性能问题转化为一致性和失效策略问题
4. 热点 key、命中率、淘汰策略、双写一致性是核心关注点

## 常见误区
- 觉得上 Redis 就一定提速
- 所有数据都缓存
- 忽视缓存一致性成本
- 把 Redis 当主存储乱用
