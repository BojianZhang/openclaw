# UNIX 网络编程 - 深摘要

## 这本书解决什么问题
它解决的是：**网络程序在底层到底是怎么通过 socket 跑起来的。**

## 核心主线
1. 套接字模型
2. 连接建立与关闭
3. 阻塞 / 非阻塞 I/O
4. 多路复用
5. 网络编程错误与边界行为

## 最关键的知识骨架
### 1. 应用层框架之下，最终都落回 socket
你在 Java 里看到的 HTTP client、Tomcat、Netty，本质最终都要回到底层连接和 I/O 行为。

### 2. 网络问题很多时候不是“业务逻辑错了”
而是：
- 连接没建好
- 超时了
- 半连接/半关闭
- 读写阻塞
- 下游没及时响应

### 3. 连接生命周期非常重要
理解 connect、accept、read、write、close 的行为，对排查 Web 问题很有帮助。

## 工程落地价值
### 适合回答的问题
- Java Web 请求为什么会卡？
- 连接超时和读取超时有什么区别？
- socket 和 HTTP 之间是什么关系？

## 与其他书的连接
- 和《TCP/IP 详解》搭配：一个偏编程接口，一个偏协议行为
