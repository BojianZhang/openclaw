# Wireshark 系列 - 深摘要

## 这组书解决什么问题
它们解决的是：**网络问题不能只靠猜，要学会抓包验证。**

## 核心主线
1. 抓包基础
2. 看懂 TCP / HTTP 报文
3. 识别重传、RST、握手失败、超时
4. 从包级证据反推应用层问题

## 最关键的知识骨架
### 1. Wireshark 的价值在于“把现象可视化”
应用日志看不出来的东西，包里往往很清楚。

### 2. 很多后端问题其实是网络链路问题
例如：
- 超时
- 连接复用异常
- TLS 握手失败
- 代理层问题
- 客户端主动断开

## 工程落地价值
### 适合回答的问题
- 为什么请求偶发超时？
- 为什么连接被重置？
- 为什么看起来服务正常但客户端一直报错？

## 与其他书的连接
- 和 HTTP 权威指南、TCP/IP 详解组合使用效果最好
