# Triage Checklist

## 第一步：先问症状
- 是启动失败，还是运行中变慢？
- 是偶发，还是持续？
- 是单机，还是全链路？
- 是报错，还是只是性能下降？

## 第二步：先问证据
- 错误日志
- 监控指标
- 时间点
- 线程栈
- GC 日志
- 慢查询日志
- 调用链 / 网关日志

## 第三步：先做分层
- Spring / 配置 / 启动
- JVM / 线程 / GC
- MySQL / Redis / 数据访问
- HTTP / TCP / 网关 / 下游调用
- 结构与职责边界

## 第四步：先走最短路径
- 只先查最可疑的一层
- 不要所有方向同时展开
