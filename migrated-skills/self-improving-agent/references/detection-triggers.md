# Detection Triggers

## 什么时候应该记录

### Corrections
当用户明确纠正你：
- “No, that's wrong...”
- “Actually...”
- “你这个不对”
- “应该是这样”

### Failures
当命令、工具、API 或外部系统失败：
- 命令返回非零
- 报错堆栈
- 连接失败
- 超时
- 输出明显异常

### Missing Capability
当用户想要你目前没有的能力：
- “为什么不能……”
- “能不能也支持……”
- “有没有办法……”

### Knowledge Gap
当你发现：
- 自己知识过时
- 文档不对
- API 行为和原理解读不一致

### Better Approach
当你发现：
- 有更稳的重复做法
- 有更省事的流程
- 有值得固定下来的最佳实践

## 记录原则
- 刚发生时就记，别拖。
- 优先记非显而易见的错误和经验。
- 一次性小失误不必写成长篇，重复问题优先记。
