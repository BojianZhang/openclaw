# Promotion Rules

## 何时升级到长期规则
当学习满足以下任一条件时，考虑升级：
- 适用于多个任务或多个文件
- 明显能防止重复犯错
- 属于项目约定或长期习惯
- 任何协作者（人或 AI）都该知道

## 升级目标
- `CLAUDE.md` / 项目指令文件：项目事实、约定、固定做法
- `AGENTS.md`：工作流、自动化、协作规则
- `.github/copilot-instructions.md`：Copilot 项目上下文
- `SOUL.md`：行为风格、交流原则
- `TOOLS.md`：工具使用 gotcha 与环境约定
- `MEMORY.md`：长期个人/项目记忆

## 升级方式
1. 先把长学习压成短规则。
2. 放进最合适的目标文件。
3. 把原条目状态改成 `promoted`。
4. 记录升级到哪里。

## 简化原则
不要把事故报告原样搬进长期文件。
长期文件里只保留：
- 该记住的规则
- 该提前避免的坑
- 该遵守的约定
