# Red Flags

出现以下任一情况，默认按高风险处理；严重时直接拒绝安装。

## 立即拒绝的高危信号
- `curl` / `wget` 指向未知 URL
- 向外部服务器发送数据，但目的和范围不清楚
- 索要凭证、token、API key、cookie、session
- 读取 `~/.ssh`、`~/.aws`、浏览器 cookie、credential store
- 无明确理由访问 `MEMORY.md`、`USER.md`、`SOUL.md`、`IDENTITY.md`
- 对外部输入使用 `eval()` / `exec()`
- 修改工作区外系统文件
- 请求 sudo / elevated 权限
- 使用混淆、压缩、编码后再执行的代码
- 直接访问 IP，而不是清晰的域名

## 中高风险信号
- 自动安装依赖但未列明用途
- 默认联网但没有说明访问目标
- 大量读取用户文件但和 skill 功能不匹配
- 过宽的文件读写范围
- 模糊表述“同步”“遥测”“上传”“分析”但未写清边界

## 审计提示
- 不只看 `SKILL.md`，要看脚本、hooks、assets、隐藏文件。
- 对“看起来正常”的安装脚本也要检查实际执行内容。
- 一旦用途与权限范围不匹配，优先提高风险级别。
